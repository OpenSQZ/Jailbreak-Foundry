# Inspect Viewer Tools

Tools for converting JBFoundry attack results into [Inspect AI](https://github.com/UKGovernmentBEIS/inspect_ai) evaluation logs for visualization.

## Overview

After running jailbreak attacks using the JBFoundry framework, these scripts convert the JSON result files into `.eval` format that can be viewed in Inspect's interactive web interface.

## Prerequisites

Install Inspect AI:
```bash
pip install inspect-ai
```

## Scripts

### 1. convert_attack_results_to_eval.py

Converts individual `results_<attack>.json` files into Inspect `.eval` logs.

**Usage:**
```bash
python convert_attack_results_to_eval.py \
    --results-dir ./results \
    --log-dir ./logs_attack_runs
```

**What it does:**
- Finds all `results_*.json` files under the specified directory
- Converts each to an Inspect-compatible `.eval` file
- Outputs to a flat directory structure
- Skips empty files automatically

**Output:**
```
logs_attack_runs/
├── attack1-model1-timestamp.eval
├── attack2-model2-timestamp.eval
└── ...
```

---

### 2. convert_attack_selector_to_eval.py

Converts `attack_selector_results.json` files into Inspect logs.

**Usage:**
```bash
python convert_attack_selector_to_eval.py \
    --input path/to/attack_selector_results.json \
    --log-dir ./logs_attack_selector
```

**What it does:**
- Processes attack selector results (multiple attacks tried per query)
- Captures all attempt details, success metrics, and timeout info
- Creates per-attempt events for detailed transcript view

---

### 3. batch_convert_attack_results.py

Batch processor that converts multiple attack result files and organizes by model.

**Usage:**
```bash
python batch_convert_attack_results.py
```

**What it does:**
- Automatically finds `results_*.json` files in `*_comprehensive` directories
- Groups outputs by victim model
- Fixed paths: reads from `results/`, writes to `results/attack_eval_by_model/`

**Output:**
```
results/attack_eval_by_model/
├── gpt-4o/
│   ├── attack1-gpt-4o-timestamp.eval
│   └── attack2-gpt-4o-timestamp.eval
├── claude-3-5-sonnet/
│   └── attack1-claude-timestamp.eval
└── ...
```

**Note:** Only processes files in directories ending with `_comprehensive`. To process all files, use `convert_attack_results_to_eval.py` instead.

---

### 4. batch_convert_selector_results.py

Batch processor for all attack selector results.

**Usage:**
```bash
python batch_convert_selector_results.py
```

**What it does:**
- Finds all `attack_selector_results.json` files under `results/`
- Organizes outputs by dataset
- Writes to `results/selector_viz/`

**Output:**
```
results/selector_viz/
├── dataset1/
│   └── selector_policy.eval
├── dataset2/
│   └── selector_policy.eval
└── ...
```

## Viewing Results

After conversion, launch the Inspect viewer:

```bash
# View attack results
inspect view --log-dir ./logs_attack_runs

# View selector results
inspect view --log-dir ./results/selector_viz

# View batch results by model
inspect view --log-dir ./results/attack_eval_by_model

# With host binding (for remote access)
inspect view --log-dir ./logs_attack_runs --host 0.0.0.0
```

The web interface provides:
- 📊 Aggregate success rates and metrics
- 🔍 Per-sample detailed views
- 💰 Token usage and cost tracking
- 📝 Attack transcripts and attempts
- 📈 Model comparison charts

## Integration with JBFoundry

### Attack Results Location

Results are generated by the JBFoundry runners:
- **Attack Runner**: `src/jbfoundry/runners/universal_attack.py`
- **Comprehensive Tests**: `src/jbfoundry/runners/test_comprehensive.py`

Results are saved to:
```
results/
├── <attack>_comprehensive/
│   └── <model>_<dataset>/
│       └── results_<attack>.json
└── test_*/
    └── selector=*/
        └── query*/
            └── results_*.json
```

### Attack Selector Results

Generated by the attack selector system, containing:
- Multiple attack attempts per query
- Success/failure for each attempt
- Token usage and cost per attempt
- Aggregate metrics across all queries

Location: `results/<test_name>/<dataset>/selector=<policy>/attack_selector_results.json`

## Troubleshooting

### No files converted

Check that your results directory contains `results_*.json` files:
```bash
find results -name "results_*.json" | head -10
```

For batch converter, ensure files are in `*_comprehensive` directories.

### Import errors

Run scripts from the `tools/inspect_viewer` directory:
```bash
cd tools/inspect_viewer
python convert_attack_results_to_eval.py --results-dir ../../results --log-dir ../../logs
```

### Empty result files

The scripts automatically skip files with no results. Check the output for "SKIPPED" messages.

### Server shuts down immediately

Ensure `.eval` files were created:
```bash
find logs_attack_runs -name "*.eval"
```

If no files exist, check for conversion errors in the script output.

## Development Notes

### Project Structure (Refactored)

The JBFoundry codebase has been reorganized:
- ✅ Main entry scripts: `src/jbfoundry/runners/`
- ✅ Setup scripts: `scripts/setup/`
- ✅ Comprehensive tests: `scripts/comprehensive_tests/`
- ❌ No longer exists: `examples/` directory

### Script Dependencies

All scripts use local imports (same directory) and standard library plus `inspect-ai`:
- No `sys.path` manipulation needed
- Run from `tools/inspect_viewer/` directory
- Import structure is simple and maintainable

### Extending

To add support for new result formats:
1. Create a new `convert_<format>_to_eval.py` script
2. Follow the pattern in existing converters
3. Use Inspect AI's `EvalLog`, `EvalSample`, `EvalResults` structures
4. Add to this README with usage examples
