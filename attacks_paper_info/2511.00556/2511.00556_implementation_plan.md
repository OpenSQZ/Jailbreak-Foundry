## 2. Attack Overview
ISA (Intent Shift Attack) is a jailbreaking strategy that transforms harmful requests into benign-appearing information-seeking queries through minimal linguistic modifications. The core hypothesis is that LLMs struggle to distinguish between malicious intent and general knowledge inquiries when the request is linguistically reframed. The attack operates in two steps: (1) normalizing the original harmful request into a standard "How to" format, and (2) applying one of five intent shifts (Person, Tense, Voice, Mood, Question) using a helper LLM to disguise the intent while preserving the semantic content.

## 4. Algorithm-to-Code Mapping
| Paper Step / Formula | Expected Behavior | Planned Code Location | Notes |
|---|---|---|---|
| 1. Normalization | Transform input query `Q` to "How to..." format using LLM. | `generate_attack` method | Use `normalization_prompt`. |
| 2. Intent Shift | Apply selected linguistic shift (e.g., Person Shift) to normalized query. | `generate_attack` method | Use prompt corresponding to `shift_type`. |
| 3. Output | Return the transformed query `Q'`. | `generate_attack` return | |

## 5. Data Flow and Control Flow
1.  **Input**: `prompt` (harmful query), `goal` (same as prompt usually), `target` (optional).
2.  **Step 1 (Normalization)**: Call `helper_model` with `normalization_prompt` + `prompt`.
    *   Result: `normalized_prompt`.
3.  **Step 2 (Shift)**: Call `helper_model` with specific `shift_prompt` (selected by `shift_type`) + `normalized_prompt`.
    *   Result: `shifted_prompt`.
4.  **Output**: `shifted_prompt`.

No loops or restarts are described in the generation process itself; it's a direct transformation.

## 7. Detailed Implementation Instructions (for Implementation Agent)

1.  **Create File**: Create `src/jbfoundry/attacks/isa_gen.py`.

2.  **Imports**: 
    - Import `ModernBaseAttack`, `AttackParameter` from `..attacks.base`.
    - Import `LLMLiteLLM` from `..llm.litellm`.
    - Import `Dict`, `Any` from `typing`.

3.  **Define Prompts (Constants)**:
    - Copy the prompts EXACTLY from `attacks_paper_info/2511.00556/ISA/utils.py` into the file as constants or a dictionary.
    - `NORMALIZATION_PROMPT`: `how_to_format_prompt` from utils.
    - `SHIFT_PROMPTS`: Dictionary mapping keys (`person`, `tense`, `voice`, `mood`, `question`) to their respective prompts from `utils.py` (`person_shift_prompt`, etc.).

4.  **Define Class `ISAGen`**:
    - Inherit from `ModernBaseAttack`.
    - Set `NAME = "isa_gen"`.
    - Set `PAPER = "2511.00556"`.

5.  **Define Parameters**:
    - `PARAMETERS` dictionary:
        - `"helper_model"`: `AttackParameter`, type `str`, default `"gpt-4o"`, cli_arg `"isa-helper"`.
        - `"shift_type"`: `AttackParameter`, type `str`, default `"question"`, choices `["person", "tense", "voice", "mood", "question"]`, cli_arg `"isa-shift"`.

6.  **Implement `__init__`**:
    - Call `super().__init__(**kwargs)`.
    - Initialize the helper LLM:
      ```python
      self.helper_llm = LLMLiteLLM.from_config(
          model_name=self.get_parameter_value("helper_model"),
          provider="openai",  # Or derive from name if needed, usually 'openai' for gpt-4o
          temperature=0.0
      )
      ```
      *Note: Ideally allow provider configuration, but for now assuming standard litellm formatted name or defaulting provider to openai is safe for "gpt-4o". If the user passes "provider/model", `from_config` handles it if we parse it, but `LLMLiteLLM` can be instantiated directly if we have the full name. Better yet, use `LLMLiteLLM.from_config` and assume the `helper_model` string might be just the model name. If it's a simple string like "gpt-4o", pass `provider="openai"`. If it has a slash, parse it.*
    - **Better approach for LLM init**:
      ```python
      helper_model = self.get_parameter_value("helper_model")
      if "/" in helper_model:
          provider, model = helper_model.split("/", 1)
      else:
          provider, model = "openai", helper_model
      
      self.helper_llm = LLMLiteLLM.from_config(
          model_name=model,
          provider=provider,
          temperature=0.0
      )
      ```

7.  **Implement `generate_attack`**:
    - **Signature**: `def generate_attack(self, prompt: str, goal: str, target: str, **kwargs) -> str:`
    - **Step 1: Normalization**:
        - Construct message: `NORMALIZATION_PROMPT + "\n\n" + prompt` (or format as suitable).
        - Call `self.helper_llm.query(msg)`.
        - Extract `normalized_text` from response.
    - **Step 2: Intent Shift**:
        - Get `shift_type` from parameters.
        - Retrieve specific prompt from `SHIFT_PROMPTS`.
        - Construct message: `shift_prompt + "\n\n" + normalized_text`.
        - Call `self.helper_llm.query(msg)`.
        - Extract `shifted_text` from response.
    - **Return**: `shifted_text`.

8.  **Error Handling**:
    - If LLM calls fail, they might raise exceptions. Let them propagate or catch and log, but `ModernBaseAttack` wrapper handles some context. The `LLMLiteLLM` raises exceptions on failure.
