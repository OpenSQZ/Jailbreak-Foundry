## 2. Attack Overview
FlipAttack is a black-box jailbreak attack that exploits the autoregressive nature of LLMs. It disguises harmful prompts by introducing "left-side noise" constructed from the prompt itself (via various flipping modes: word order, characters in word, characters in sentence). It then uses a "flipping guidance module" (system prompts, Chain-of-Thought, LangGPT, few-shot demonstrations) to guide the LLM to decode, understand, and execute the hidden harmful instruction. The attack is single-turn and relies on prompt engineering rather than iterative optimization.

## 5. Data Flow and Control Flow
1.  **Input**: `prompt` (harmful goal), `flip_mode`, `cot`, `lang_gpt`, `few_shot`.
2.  **Processing**:
    *   Determine `mode_prompt` string based on `flip_mode`.
    *   Determine `cot_prompt` string based on `cot`.
    *   Construct `system_prompt` (Standard or LangGPT variant).
    *   Flip the `prompt` using the selected mode to get `disguised_prompt`.
    *   Construct `user_prompt` starting with `TASK is '{disguised_prompt}'`.
    *   If `few_shot` is True, generate 3 examples (first half, "Follow my instruction", second half) and append to prompts.
    *   If model is Llama (check `self.args.model`), sanitize system prompt strings.
3.  **Output**: A list of message dictionaries `[{"role": "system", "content": ...}, {"role": "user", "content": ...}]` returned by `generate_attack`.
