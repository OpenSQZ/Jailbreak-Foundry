## 2. Attack Overview
**Human-like Psychological Manipulation (HPM)** is a black-box, multi-turn jailbreak attack that exploits "over-optimized social priors" in LLMs. Instead of brute-forcing prompts, HPM first profiles the target model's latent psychological traits (using the Five-Factor Model) to identify vulnerabilities (e.g., high Agreeableness). It then selects a tailored "semantic anchor" (e.g., Authority Intimidation, Gaslighting) based on a susceptibility matrix and executes a hierarchical attack plan using an Attacker Agent to induce a "compliance-safety decoupling," forcing the model to prioritize persona consistency over safety.

## 5. Data Flow and Control Flow
1.  **Input**: `goal` (malicious objective), `target_model` (reference).
2.  **Profiling**: 
    *   Generate/Load probes $Q$ (Templates from Appendix B).
    *   `target_llm.query(Q)` -> Responses $A$.
    *   `judge_llm` evaluates $A$ -> Profile vector $V_p$ (O, C, E, A, N scores).
3.  **Strategy Selection**:
    *   Determine dominant trait $d^*$.
    *   Lookup optimal anchor $S^*$ in Matrix $W$.
4.  **Attack Execution**:
    *   Initialize `history` with system prompt (SPC) if applicable, or just start dialogue.
    *   **Loop** `t` from 1 to `max_turns`:
        *   Attacker Agent generates utterance $u_t$ based on plan/strategy.
        *   Target Model generates response $r_t$.
        *   Update `history`.
        *   If `r_t` is jailbroken (Judge/Classifier), **break** and return `history`.
5.  **Output**: The full conversation history (List[Dict]) that achieved the jailbreak.

## 7. Detailed Implementation Instructions

1.  **Imports & Setup**:
    - Import `ModernBaseAttack`, `AttackParameter`.
    - Import `LLMLiteLLM`, `RichResponse`.
    - Import `json`, `re`, `numpy` (for argmax).

2.  **Class Definition**:
    - Define `HPMAttack(ModernBaseAttack)`.
    - Set `NAME = "hpm_gen"`.
    - Define `PARAMETERS` matching Section 3.
    - **Hardcode Data**:
        - `SUSCEPTIBILITY_MATRIX`: Dict mapping personality keys (e.g., 'O+', 'A-') to Dict of strategy scores (e.g., {'GL': 0.2, 'AI': 0.5}).
        - `PROBE_TEMPLATES`: Dict mapping traits (O, C, E, A, N) to High/Low probe templates (from Table 8).
        - `STRATEGY_NAMES`: Map abbreviations (GL, AI) to full names (Gaslighting, Authority Intimidation).

3.  **Initialization (`__init__`)**:
    - Call `super().__init__`.
    - Initialize `self.attacker_llm`, `self.target_llm`, `self.judge_llm` using `LLMLiteLLM.from_config`. Use provided arguments.

4.  **Profiling Implementation (`_profile_target`)**:
    - Create a list of probes (e.g., 2 High, 2 Low per dimension = 20 queries, or use user arg).
    - **Optimization**: Run probes in parallel or batch if possible.
    - Collect responses from `self.target_llm`.
    - **Scoring**: For each dimension, send (Probe, Response) to `self.judge_llm` using "Implicit Behavioral Probes" rubric (Appendix A.1).
    - Compute average score for each trait (1-5).
    - Return `profile` (Dict[str, float]).

5.  **Strategy Selection (`_select_strategy`)**:
    - Find trait $d$ with max deviation: `max(|score - 3.0|)`.
    - Determine polarity (e.g., if O=4.5 -> 'O+', if O=1.2 -> 'O-').
    - Look up row in `SUSCEPTIBILITY_MATRIX`.
    - Return strategy with highest score in that row.

6.  **Attacker Prompt Construction**:
    - Implement `_get_attacker_system_prompt`.
    - Use "Prompt Template for HPM Plan Generation" (Appendix B).
    - Inject `malicious_goal`, `victim_personality_profile`, `inferred_vulnerabilities`, `primary_manipulation_strategy`.

7.  **Generate Attack (`generate_attack`)**:
    - **Step 1: Profile**. Call `_profile_target`. (Log results).
    - **Step 2: Select Strategy**. Call `_select_strategy`. (Log selected strategy).
    - **Step 3: Initialize Dialogue**.
    - **Step 4: Loop**:
        - Construct message for Attacker: "Current history: ... Generate next utterance to achieve goal using strategy...".
        - Query `attacker_llm`. Parse JSON (if structured) or text to get `utterance`.
        - Append `utterance` to `conversation_history` (Role: User).
        - Query `target_llm` with `conversation_history`.
        - Append `response` to `conversation_history` (Role: Assistant).
        - **Success Check**: Simple heuristic (refusal check) or optional Judge call. If successful, break.
    - **Return**: `conversation_history` (List of dicts). `universal_attack.py` will handle the final query/logging, but since this is a multi-turn history, `universal_attack` needs to handle a list. `LLMLiteLLM.query` supports list of messages.

8.  **Constraints & Error Handling**:
    - Handle JSON parsing errors from Attacker Agent.
    - If profiling fails (e.g., API error), fallback to "Base" profile.
    - Ensure `conversation_history` format matches `LLMLiteLLM` expectations (`{"role": "...", "content": "..."}`).

## 8. Validation and Coverage Plan
- **Profiling Check**: Verify that `_profile_target` produces valid 1-5 scores.
- **Matrix Check**: Verify strategy selection logic correctly picks max value from matrix row.
- **Integration Check**: Ensure `generate_attack` returns a structure compatible with `universal_attack.py` (which expects the return value to be passed to `llm.query`). A list of messages is valid.
- **Cost**: HPM is expensive (Probes + Multi-turn). Ensure `CostTracker` is passed if possible, or accept `LLMLiteLLM` internal tracking.
