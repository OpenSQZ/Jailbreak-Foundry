# Blueprint for Implementing the ICE Jailbreak Attack on LLMs

This blueprint outlines the implementation of the ICE (Intent Concealment and divErsion) jailbreak attack on large language models (LLMs) as described in the paper. The ICE attack is designed to bypass LLM safety mechanisms by decomposing malicious queries into hierarchical fragments and augmenting them with semantic expansions. The attack is executed in a black-box setting, where the attacker can only query the model and observe its outputs.

## Overview

The ICE attack consists of four main components:

1. **Hierarchical Split**: Decomposes the input prompt into hierarchical fragments based on syntactic dependencies.
2. **Semantic Expansion**: Augments the prompt with semantically related words to obfuscate the attack intent.
3. **Reasoning Mask**: Combines hierarchical fragments and semantic expansions into a dispersed sentence representation.
4. **Environmental Construction**: Embeds the transformed prompt within an inference or question-answering framework.

## Implementation Details

### 1. Hierarchical Split

**Purpose**: Decompose the input prompt into hierarchical fragments to conceal the attack intent.

**Inputs**:
- `S`: Original prompt as a list of words.
- `G`: Dependency graph of the sentence, with edges representing syntactic dependencies.

**Outputs**:
- `L`: List of tuples containing words and their updated hierarchical levels.

**Procedure**:
1. Identify non-root verbs with both parent and child nodes.
2. Randomly select a subset of these verbs and modify their dependency relations.
3. Increment the hierarchical level of selected verbs and propagate changes to child nodes.
4. Define breakpoints for hierarchical splitting based on level changes and preserved dependencies.
5. Normalize levels to contiguous natural numbers.

**Pseudocode**:
```python
def hierarchical_split(S, G):
    W_verbs = {w for w in S if pos(w) == 'VERB' and has_parent_and_child(w, G)}
    W_mod = random_subset(W_verbs)
    for w in W_mod:
        modify_dependencies(w, G)
        increment_level(w)
    B = define_breakpoints(S, G)
    P = pair_breakpoints(B)
    for (b_s, b_e) in P:
        increment_levels(S, b_s, b_e)
    L = normalize_levels(S)
    return L
```

### 2. Semantic Expansion

**Purpose**: Augment the prompt with semantically related words to enhance obfuscation.

**Inputs**:
- `S`: Original prompt as a list of words.

**Outputs**:
- `E`: Set of semantically expanded words.

**Procedure**:
1. Construct verb and noun sets from the prompt.
2. Use DistilBERT to predict the sentiment label of the prompt.
3. Retrieve semantically related words for a randomly selected verb and noun using WordNet.
4. Analyze toxicity and generate descriptive words for the most toxic word.

**Pseudocode**:
```python
def semantic_expansion(S):
    V, N = construct_verb_noun_sets(S)
    e_s = predict_sentiment(S)
    r_v = random_related_word(V)
    t_n = representative_noun_phrase(N)
    O, d = analyze_toxicity(V, N)
    E = {e_s, r_v, t_n} | O | {d}
    return E
```

### 3. Reasoning Mask

**Purpose**: Combine hierarchical fragments and semantic expansions into a dispersed sentence representation.

**Inputs**:
- `L`: List of words with hierarchical levels.
- `E`: Set of semantic expansions.

**Outputs**:
- `S'`: Dispersed sentence representation.

**Procedure**:
1. Group words into segments based on hierarchical levels and replace with placeholders.
2. Generate explanation sentences for each placeholder and semantic expansion.
3. Randomly order and concatenate explanation sentences.

**Pseudocode**:
```python
def reasoning_mask(L, E):
    segments = group_segments(L)
    placeholders = assign_placeholders(segments)
    explanation_sentences = generate_explanations(placeholders, E)
    S' = concatenate_explanations(explanation_sentences)
    return S'
```

### 4. Environmental Construction

**Purpose**: Embed the transformed prompt within an inference or question-answering framework.

**Inputs**:
- `S'`: Dispersed sentence representation.

**Outputs**:
- `final_prompt`: Complete prompt for querying the LLM.

**Procedure**:
1. Prepend and append predefined prompts to disguise the task.
2. Construct the final prompt for either question-answering or text generation scenarios.

**Pseudocode**:
```python
def environmental_construction(S', scenario):
    if scenario == 'QA':
        final_prompt = f"Reconstruct the sentence: {S'} and answer the question."
    elif scenario == 'TextGen':
        final_prompt = f"Reconstruct the sentence: {S'}."
    return final_prompt
```

## Execution Flow

1. **Input**: Original prompt `S`.
2. **Hierarchical Split**: Decompose `S` into hierarchical fragments.
3. **Semantic Expansion**: Augment `S` with semantic expansions.
4. **Reasoning Mask**: Combine fragments and expansions into `S'`.
5. **Environmental Construction**: Embed `S'` into a complete prompt for querying the LLM.
6. **Output**: Query the LLM with the final prompt and observe the response.

## Model Roles

- **DistilBERT**: Used for sentiment analysis in the Semantic Expansion step.
- **WordNet**: Used for retrieving semantically related words.
- **LLM (e.g., GPT-4o)**: Used for analyzing toxicity and generating descriptive words.

## Attack Artifacts

- **Decoding Settings**: Single-query execution to minimize time consumption.
- **Filtering/Heuristics**: Use of semantic expansions and hierarchical splits to obfuscate intent.
