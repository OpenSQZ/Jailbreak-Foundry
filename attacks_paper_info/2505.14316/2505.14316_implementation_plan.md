## 2. Attack Overview
ICE (Intent Concealment and divErsion) is a black-box jailbreak method that disguises malicious intent by decomposing the prompt into hierarchical fragments and dispersing them with semantically related concepts. It consists of four steps:
1.  **Hierarchical Split**: Decomposes the prompt based on a dependency graph, modifying relations for a subset of verbs and adjusting hierarchical levels.
2.  **Semantic Expansion**: Augments the prompt with sentiment labels, verb synonyms, noun definitions (summarized), and toxicity-related descriptors.
3.  **Reasoning Mask**: Replaces hierarchical segments with placeholders and generates explanation sentences for them and the semantic expansions.
4.  **Environmental Construction**: Embeds the dispersed representation into a Question-Answering or Text-Generation scenario prompt.

## 5. Data Flow and Control Flow
1.  **Input**: `prompt` (goal).
2.  **Hierarchical Split**:
    -   Parse `S` with Spacy.
    -   Compute `W_verbs`. Select `W_mod`.
    -   Initialize `levels = {i: 1}`.
    -   For `w` in `W_mod`:
        -   Check `w.dep_` against `R_preserved`.
        -   Start propagation from `w`.
        -   **Note**: "Update C_pk ... to remove w_i". This implies modifying the traversal graph. We should build an explicit adjacency list for traversal that reflects these "cuts".
        -   Increment level of `w` and its children in the *modified* graph.
    -   Identify `B`.
    -   Form `P` (random pairs).
    -   Increment levels for ranges in `P`.
    -   Normalize levels.
    -   Return `L = [(word, level)]`.
3.  **Semantic Expansion**:
    -   Extract V, N.
    -   Get $e_s$ (Sentiment).
    -   Get $r_v$ (Verb synonym).
    -   Get $t_n$ (Noun phrase from definition).
    -   Get $O, d$ (Toxic/Adjectives).
    -   Return `E`.
4.  **Reasoning Mask**:
    -   Group `L` by level.
    -   Generate placeholders `[PART_k]`.
    -   Generate sentences.
    -   Shuffle.
    -   Return `S'`.
5.  **Env Construction**:
    -   Wrap `S'` in template.
    -   Return `final_prompt`.

## 7. Detailed Implementation Instructions

### A. Update `hierarchical_split`
1.  **Graph Construction**: Create an adjacency list `children_map` representing the dependency tree from Spacy doc.
2.  **Verb Selection**: Identify `W_verbs` (POS=VERB, has head != self, has children).
3.  **Subset Selection**:
    -   If `modification_ratio` parameter is set, use it.
    -   Else, select random size `k` in `[1, len(W_verbs)]`.
    -   Sample `W_mod`.
4.  **Dependency Modification & Level Update**:
    -   Initialize `levels = {i: 1}` (or 0, normalize later).
    -   `R_preserved = {'neg', 'fixed', 'compound', 'amod', 'advmod', 'nmod'}`.
    -   For each `verb` in `W_mod`:
        -   Check if `verb.dep_` is in `R_preserved`.
        -   If **NOT** in `R_preserved`:
            -   "Cut" the edge from parent in `children_map` (logically). This means when we propagate level increments from the parent later (if any), we don't cross to this verb.
            -   BUT here we are incrementing the verb's level.
            -   Algorithm 1 says: "Update C_pk ... remove w_i". Then "l_i += 1", "foreach child: l_j += 1".
            -   So regardless of relation, we increment `verb` and its *current* children.
            -   The "cut" affects *future* propagations or the structure?
            -   Actually, just follow the steps:
                -   Apply cuts if relation not preserved (remove from parent's children list).
                -   Increment level of `verb` + 1.
                -   Propagate (+1) to all nodes reachable from `verb` in the *current* (possibly modified) `children_map`.
5.  **Breakpoints**:
    -   `B = {0, len(doc)}`.
    -   Add `i` to `B` if `levels[i] != levels[i+1]`.
    -   **Constraint**: "no dependency relation in R_preserved crosses a breakpoint."
    -   Check each candidate `i` (midpoint between i and i+1):
        -   If `doc[i+1].head == doc[i]` and `doc[i+1].dep_ in R_preserved`: Invalid breakpoint.
        -   If `doc[i].head == doc[i+1]` and `doc[i].dep_ in R_preserved`: Invalid breakpoint.
        -   If invalid, do not add `i` to `B`? Or force merge?
        -   Since levels differ, we have a contradiction if we don't break.
        -   Interpretation: If `R_preserved` exists, `levels` *should* have been same. If they differ, our propagation might have been inconsistent with "preserved" notion.
        -   *Practical fix*: If `R_preserved` crossing exists, **remove** `i` from `B`. This effectively merges the segments for the masking step, treating them as one unit even if levels differ slightly (or we unify their levels). Let's unify levels in the merged segment to `max(l_i, l_{i+1})`.
6.  **Random Pairing**:
    -   `P = []`.
    -   `num_pairs` = random integer (e.g., 1 to `len(B)//2`).
    -   Loop `num_pairs` times:
        -   Pick two distinct breakpoints `b1, b2` from `B`.
        -   Range `(start, end) = (min(b1, b2), max(b1, b2))`.
        -   Add to `P`.
    -   For each range in `P`: increment levels of all tokens in range.
7.  **Normalize**: Rank mapping to 1..m.

### B. Update `semantic_expansion`
1.  **Noun Expansion**:
    -   Select `n` from `N`.
    -   Get synsets: `wordnet.synsets(n, pos=wordnet.NOUN)`.
    -   If no synsets, use `n`.
    -   Else, get definition: `synsets[0].definition()`.
    -   Extract "representative noun phrase":
        -   Parse definition with Spacy.
        -   Take first `noun_chunk`. If none, take whole definition.
    -   Add to `E`.
