## 2. Attack Overview
**JailExpert** is an automated jailbreak framework that leverages past attack experiences (successful jailbreak prompts from other methods) to generate new attacks. It formalizes experiences into a structure containing mutations, templates, and success/failure counts. It groups these experiences based on "jailbreak semantic drift" (the semantic difference between the initial query and the jailbroken prompt) using K-Means clustering.

For a new target query, JailExpert:
1.  Ranks experience clusters by comparing the target's potential semantic drift with cluster centers.
2.  Iteratively attempts attacks using the **Representative Pattern** of a cluster (most frequent/successful strategy).
3.  If that fails, it searches within the cluster for the specific **Experience** most similar to the target query and attempts that.
4.  (Optional/Best Effort) Dynamically updates experience stats (success/failure) - *Note: Dynamic updates during the attack loop are limited by the `ModernBaseAttack` stateless interface, so we will implement the "Static" retrieval strategies which are the core contribution.*

## 5. Data Flow and Control Flow

1.  **Initialization**:
    *   Load `experiences_examples.json`.
    *   Batch compute embeddings for all `pre_query` (I) and `full_query` (J) in experiences.
    *   Compute Drift Vectors `D = Emb(J) - Emb(I)`.
    *   Perform K-Means clustering on `D`.
    *   Identify Representative Pattern `(Mut_rep, Temp_rep)` for each cluster.
    *   Build local search index (simple list or nearest neighbor) for each cluster.

2.  **`generate_attack(prompt, attempt_index, ...)`**:
    *   **State Check**: If `prompt` != `self.current_prompt`, reset state.
        *   Embed `prompt`.
        *   Compute "projected drift" for each cluster using its Representative Pattern applied to `prompt`.
        *   Rank clusters by similarity of projected drift to cluster center.
        *   Store `sorted_clusters`.
    *   **Action Selection**:
        *   Map `attempt_index` to an action `(ClusterID, Type)`.
        *   Pattern: `[ (C1, 'REP'), (C1, 'EXP'), (C2, 'REP'), (C2, 'EXP'), ... ]`
    *   **Generation**:
        *   If **REP**: Apply Representative Mutation + Template to `prompt`.
        *   If **EXP**: Search cluster `C` for experience `E` maximizing `Sim(Prompt, E.pre_query) * SuccessWeight`. Apply `E.mutation` + `E.method` to `prompt`.
    *   **Return**: The constructed jailbreak prompt.

## 7. Detailed Implementation Instructions

1.  **Imports**: Include `numpy`, `sklearn.cluster.KMeans`, `sklearn.metrics.pairwise.cosine_similarity`, `json`, `os`, `litellm`.
2.  **Mutation Classes**: Port the `MutationBase` and subclasses from `codes/mutation.py` as inner classes or a concise `MutationEngine` class.
    *   Implement: `Base64`, `Rot13`, `Leetspeak`, `Disemvowel`.
    *   Implement LLM-based: `Rephrase`, `Artificial`, `Misspell`, `Translation`, etc. using `litellm.completion`.
    *   **Constraint**: Ensure LLM calls for mutations are robust (try/except).
3.  **Data Loading**:
    *   In `__init__`, load the JSON.
    *   If file missing, log warning and use a minimal hardcoded fallback (e.g., 1-2 generic templates) to prevent crash, or raise error if critical.
4.  **Embedding & Indexing**:
    *   Implement `get_embedding(text)` using `litellm.embedding`.
    *   In `__init__`, compute embeddings. **Cache** these if possible (check for `cached_embeddings.pkl`).
    *   Run KMeans. Handle `n_clusters` logic (if experiences < n_clusters, reduce K).
    *   Store `cluster_centers` and `cluster_assignments`.
5.  **Ranking Logic**:
    *   Implement `_rank_clusters(prompt)`:
        *   For each cluster $i$:
            *   Get Rep Pattern $A_i$.
            *   Generate candidate $J_{cand} = A_i(prompt)$.
            *   Get $V_{cand} = Emb(J_{cand})$.
            *   $V_{prompt} = Emb(prompt)$.
            *   $\Delta_{cand} = V_{cand} - V_{prompt}$.
            *   Score = CosineSim($\Delta_{cand}$, $Center_i$).
        *   Sort by Score descending.
6.  **Search Logic**:
    *   Implement `_search_experience(cluster_id, prompt)`:
        *   Get all experiences in cluster.
        *   Scores = CosineSim($Emb(prompt)$, $Emb(Exp.pre\_query)$) * SuccessRateAdjustment.
        *   Return top experience.
7.  **`generate_attack`**:
    *   Handle `attempt_index` mapping.
    *   Use `self.current_prompt` to detect new queries.
    *   Return the final string.

**Note on "Dynamic Update"**: Since we cannot easily get feedback in `generate_attack` (it returns string, doesn't wait for result), we will implement the **Static** version. The paper shows this is highly effective. We will log a message that dynamic updates are disabled in this integration.
