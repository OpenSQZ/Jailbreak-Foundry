### Blueprint for SATA Implementation

#### Overview:

The SATA paradigm consists of masking harmful keywords in a query and creating assistive tasks to encode those keywords implicitly or explicitly. Two main methods are used: Masked Language Model (MLM) and Element Lookup by Position (ELP). This blueprint will cover the entire process from preprocessing through execution and final evaluation.

#### Components:

1. **Masking Harmful Keywords:**
   - **Input:** A harmful instruction/query string.
   - **Output:** A masked query with harmful keywords replaced by `[MASK]` tokens.
   - **Dependencies:** Uses GPT-40 for identifying harmful keywords.
   - **Granularity:** Single word (SW), single phrase (SP), multiple words (MW), multiple phrases (MP).

2. **Assistive Tasks:**
   - **Masked Language Model (MLM) Task:**
     - **Input:** A masked query.
     - **Output:** A paraphrased instruction with `[MASK]` tokens surrounded by synthesized context.
     - **Dependencies:** Uses GPT-3.5-turbo for paraphrasing; GPT-40 to synthesize context from wiki entries.
   
   - **Element Lookup by Position (ELP) Task:**
     - **Input:** A masked query.
     - **Output:** A query that maps list elements to `[MASK]` tokens using victims LLM for element identification.
     - **Dependencies:** Commendatory words list generated by GPT-3.5-turbo; victim LLM for position matching.

3. **Execution and Evaluation:**
   - **Attack Execution:**
     - **Input:** Completed prompts from MLM or ELP tasks.
     - **Output:** Harmful response from victim LLM.
     - **Evaluation:** Measures of Harmful Score (HS) and Attack Success Rate (ASR) judged by GPT-40.

4. **Defensive Measures:**
   - **Setup Procedures:** Implement paraphrasing, self-reminder prompts, and RAG (Retrieval-Augmented Generation) for defense.
   - **Detection:** Use PPL-filter to hide adversarial prompts.

5. **Optional Considerations:**
   - **Configuration Variants:** Different masking granularities and task configurations (top-1 vs. ensemble) for performance optimization.

3. **Evaluation:**

   - `evaluate_results(responses)`: Aggregate results for performance statistics.

   ```python
   def evaluate_results(responses):
       total_successes = 0
       hs_scores = []
       for response in responses:
           hs, asr = evaluate_response(response)
           hs_scores.append(hs)
           if asr == 1:
               total_successes += 1
       average_hs = sum(hs_scores) / len(hs_scores)
       average_asr = total_successes / len(responses) * 100
       return average_hs, average_asr
   ```
