## 2. Attack Overview
Attack via Implicit Reference (AIR) is a jailbreak method that exploits the in-context learning capabilities of LLMs. It operates in two stages:
1.  **Prompt Rewriting**: The malicious objective is decomposed into a structured outline with nested benign objectives (e.g., an abstract, background) and the malicious behavior is referenced implicitly (e.g., "the content in paragraph K") to bypass refusal filters.
2.  **Refinement**: After the target model generates the initial response (which includes the benign context and the malicious content), a follow-up request is sent to expand on the specific malicious paragraph, removing the benign context and judgments.

The core innovation is using "implicit references" to hide the malicious intent within a benign structural context.

## 5. Data Flow and Control Flow
1.  **Input**: `generate_attack(prompt, goal, target, **kwargs)`
2.  **Rewrite**:
    -   Select template based on `attack_method`.
    -   Query Rewrite LLM -> `outline`.
    -   Construct `request_reformulated` = Instruction + Outline + Constraint.
3.  **Turn 1 (Internal)**:
    -   Instantiate Target LLM (using `self.args`).
    -   Query Target LLM with `request_reformulated`.
    -   Receive `response_1`.
4.  **Turn 2 (Preparation)**:
    -   Construct `follow_up_prompt` based on `attack_method` (selecting correct paragraph index).
    -   Construct `messages` list:
        -   `{"role": "user", "content": request_reformulated}`
        -   `{"role": "assistant", "content": response_1}`
        -   `{"role": "user", "content": follow_up_prompt}`
5.  **Output**: Return `messages` (List[Dict]). The framework will execute this as a multi-turn conversation, generating the final response.
