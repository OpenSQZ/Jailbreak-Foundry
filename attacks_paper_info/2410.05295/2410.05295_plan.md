### Blueprint for AutoDAN-Turbo

#### 1. Overview

AutoDAN-Turbo is a black-box jailbreak method designed to autonomously discover and exploit jailbreak strategies to attack large language models (LLMs). The framework consists of three main modules:

- **Attack Generation and Exploration Module**
- **Strategy Library Construction Module**
- **Jailbreak Strategy Retrieval Module**

Each module plays a specific role in the lifecycle of the attack, from generating prompts to evaluating and storing effective strategies.

#### 2. Modules and Functions

##### 2.1 Attack Generation and Exploration Module

**Purpose**: Generate jailbreak prompts, interact with the target LLM, and evaluate responses.

- **Inputs**: Malicious request `M`, strategy list `Γ`, attacker LLM, target LLM, scorer LLM.
- **Outputs**: Attack logs containing prompts, responses, and scores.

**Functions**:

1. **`generate_attack_prompt(goal, strategies_list)`**
   - **Purpose**: Generate a prompt for the attacker LLM based on the goal and strategies.
   - **Inputs**: `goal` (malicious request), `strategies_list` (retrieved strategies).
   - **Outputs**: Attack prompt `P`.
   - **Logic**:
     - If `strategies_list` is empty, generate a prompt without strategy constraints.
     - If `strategies_list` contains effective strategies, incorporate them into the prompt.
     - If `strategies_list` contains ineffective strategies, instruct the attacker to avoid them.

2. **`evaluate_response(prompt, response)`**
   - **Purpose**: Evaluate the target LLM's response to the attack prompt.
   - **Inputs**: `prompt` (attack prompt), `response` (target LLM's response).
   - **Outputs**: Score `S` (1-10 scale).
   - **Logic**: Use predefined criteria to score the response based on its alignment with malicious intent.

##### 2.2 Strategy Library Construction Module

**Purpose**: Build and update a library of effective jailbreak strategies.

- **Inputs**: Attack logs, summarizer LLM.
- **Outputs**: Updated strategy library.

**Functions**:

1. **`extract_strategies_from_logs(attack_logs)`**
   - **Purpose**: Extract and summarize strategies from attack logs.
   - **Inputs**: `attack_logs` (list of tuples containing prompts, responses, and scores).
   - **Outputs**: New strategies in JSON format.
   - **Logic**:
     - Compare pairs of logs to identify improvements.
     - Use the summarizer LLM to generate strategy descriptions and definitions.

2. **`update_strategy_library(new_strategies)`**
   - **Purpose**: Add new strategies to the strategy library.
   - **Inputs**: `new_strategies` (list of strategies in JSON format).
   - **Outputs**: Updated strategy library.
   - **Logic**: Check for duplicates and add unique strategies.

##### 2.3 Jailbreak Strategy Retrieval Module

**Purpose**: Retrieve effective strategies from the library for use in generating attack prompts.

- **Inputs**: Response embedding `ER`, strategy library.
- **Outputs**: Retrieved strategies `Γ`.

**Functions**:

1. **`retrieve_strategies(response_embedding)`**
   - **Purpose**: Retrieve strategies based on the similarity of response embeddings.
   - **Inputs**: `response_embedding` (embedding vector of the response).
   - **Outputs**: List of strategies `Γ`.
   - **Logic**:
     - Compare `response_embedding` with keys in the strategy library.
     - Select top strategies based on similarity and score differences.

#### 3. Execution Flow

1. **Initialization**:
   - Load initial strategy library (if available).
   - Set parameters: maximum iterations `T`, termination score `ST`.

2. **Warm-up Exploration**:
   - For each malicious request, generate attack prompts without strategies.
   - Collect attack logs and extract initial strategies.

3. **Lifelong Learning**:
   - For each request, generate prompts using retrieved strategies.
   - Evaluate responses and update the strategy library with new findings.

4. **Testing**:
   - Use the fixed strategy library to generate prompts and evaluate responses.
   - Measure attack success rates and adjust strategies as needed.

#### 4. Prompts and Templates

- **Attacker LLM Prompts**:
  - Initial prompt without strategies.
  - Prompt with effective strategies.
  - Prompt avoiding ineffective strategies.

- **Scorer LLM Prompt**:
  - Detailed scoring criteria for evaluating responses.

- **Summarizer LLM Prompt**:
  - Instructions for summarizing and defining new strategies.

#### 5. Model Roles

- **Attacker LLM**: Generates jailbreak prompts.
- **Target LLM**: Provides responses to prompts.
- **Scorer LLM**: Evaluates responses and assigns scores.
- **Summarizer LLM**: Extracts and defines new strategies.
