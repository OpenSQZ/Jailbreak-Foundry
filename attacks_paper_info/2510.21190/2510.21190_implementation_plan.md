## 2. Attack Overview
**TrojFill** is a black-box jailbreak attack that reframes unsafe instructions as a "template filling with unsafety reasoning" task. It uses an attacker LLM to:
1.  Extract unsafe terms from the malicious instruction and replace them with placeholders (creating a "moderate instruction").
2.  Identify the text type (e.g., tutorial, email).
3.  Obfuscate the unsafe terms (using Caesar cipher, Base64, or text splitting).
4.  Embed the moderate instruction and obfuscated terms into a specific template that asks the model to first generate a detailed example (the jailbreak) and then analyze its unsafety.
5.  Iteratively rewrite the moderate instruction using the attacker LLM to diversify the attack if the initial attempt fails.

## 5. Data Flow and Control Flow
- **Inputs to `generate_attack`**: `prompt` (malicious goal), `attempt_index` (current iteration).
- **State Management**: The attack requires history of previous rewrites to generate the next one (Algorithm 1, line 12-13). Since `universal_attack.py` re-calls `generate_attack` for each attempt, the class must maintain a state dictionary `self.history` keyed by `prompt` to store the conversation history for that specific goal.
- **Concurrency**: A `threading.Lock` will protect `self.history` to ensure thread safety when running multiple examples in parallel.
- **Control Flow**:
    -   **Attempt 1**: Run extraction, text type recognition, transformation, and template filling. Initialize history.
    -   **Attempt > 1**: Retrieve history. Run rewrite (using Attacker LLM), transformation, and template filling. Update history.

3.  Import `threading`, `re`, `random`, `base64`.
4.  Define the **Prompts** as constants (copied from `Attacker.py`):

    -   `TRANSFORM_PROMPT` (Initial extraction)
    -   `REWRITE_PROMPT`
    -   `TROJFILL_TEMPLATE`

### Class `TrojFillGen`
1.  **`__init__`**:
    -   Initialize `self.history = {}` and `self.lock = threading.Lock()`.
    -   Initialize the attacker LLM: `self.attacker_llm = LLMLiteLLM.from_config(model_name=self.get_parameter_value("attacker_model"), provider="openai")` (defaulting provider to openai or inferring, but framework handles this. Actually better to use `attacker_model` parameter and allow user to specify provider if needed, but for simplicity assume standard provider string format like `provider/model` or just `model` if mapped). *Correction*: Use `model_name` and let `LLMLiteLLM` handle it, or add an `attacker_provider` parameter. Let's add `attacker_provider` defaulting to "openai" (or "gemini" if the default model is gemini).
    -   Call `super().__init__`.

2.  **Helper Methods** (from `utils.py`):
    -   `_caesar_cipher(text, shift=1)`
    -   `_split_text_random(text)`
    -   `_encode_base64(text)`
    -   `_transform_keywords(harmful_words, replaced_words, transform_type)`: Generates the `(replace <placeholder> with <obfuscated>, ...)` string.

3.  **LLM Interaction Methods** (from `Attacker.py`):
    -   `_extract_and_replace(prompt)`: Calls Attacker LLM with `TRANSFORM_PROMPT`. Parses output to get `harmful_words`, `replaced_words`, `replaced_prompt`, and the full `response`.
    -   `_get_text_type(prompt)`: Calls Attacker LLM with `TEXT_TYPE_PROMPT`. Parses "Type: ...".
    -   `_rewrite_instruction(prompt, history)`: Appends `REWRITE_PROMPT` to history. Calls Attacker LLM. Parses output to get new `harmful_words`, `replaced_words`, `replaced_prompt`. Updates history.

4.  **`generate_attack(prompt, goal, target, attempt_index, ...)`**:
    -   **Locking**: Use `with self.lock:` to safely access/modify `self.history`.
    -   **Check Key**: Key is `prompt` (the goal).
    -   **Attempt 1 Logic**:
        -   Call `_extract_and_replace(prompt)`.
        -   Call `_get_text_type(replaced_prompt)`.
        -   Initialize history list with `[User(TransformPrompt), Model(TransformResponse)]`.
        -   Store `text_type`, `harmful_words`, `replaced_words`, `replaced_prompt` in `self.history[prompt]`.
    -   **Attempt > 1 Logic**:
        -   If `prompt` not in history (e.g. restart or error), fall back to Attempt 1 logic.
        -   Else, retrieve history.
        -   Call `_rewrite_instruction`.
        -   Update `self.history[prompt]` with new values and append to conversation history.
    -   **Assembly**:
        -   Get `harmful_words`, `replaced_words`, `replaced_prompt`, `text_type` from current state.
        -   Call `_transform_keywords` to build the substitution string (e.g., `replace <obj> with <encoded>`).
        -   Fill `TROJFILL_TEMPLATE` with `transformed_prompt` (the substitution string appended to the moderate instruction) and `text_type`.
    -   **Return**: The filled template string.

### Parsers
-   **Extraction Parser**: Regex for `harmful word \d+: (.+) -> <(.+)>` and `replaced prompt: (.+)`. (See `Attacker.py:prompt_transform`).
-   **Text Type Parser**: Regex for `Type:\s*(.*)`.
