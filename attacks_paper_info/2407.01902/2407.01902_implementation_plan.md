## 2. Attack Overview
SeqAR (SEQuential Auto-generated characters) is a black-box jailbreak attack that optimizes a universal template containing multiple role-playing characters. Unlike single-character attacks, SeqAR generates a sequence of characters that "speak" sequentially to the target LLM, increasing distraction and attack success. The optimization process uses an "Attacker LLM" to generate character descriptions and a "Judgement Model" to score them on a small training set of malicious queries. The characters are optimized greedily one by one (fixing previous characters) to maximize the attack success rate.

## 4. Algorithm-to-Code Mapping

| Paper Step / Formula | Expected Behavior | Planned Code Location | Notes |
|---|---|---|---|
| Initialize | Load simple start character, init empty sequence. | `optimize()` | |
| Loop $r=1$ to $R$ | Iterate to add characters one by one. | `optimize()` | |
| Loop $i=1$ to $I$ | Generate $I$ candidate characters for current slot. | `optimize()` | |
| Character Generation | $X \sim LLM_{att}(T_{meta})$ | `_generate_character()` | Uses Meta Prompt (Fig 8). |
| Response Generation | $D_r \leftarrow LLM_{trg}(D, X, T_{eval})$ | `_get_target_responses()` | Apply current template to training set $D$. |
| Character Scoring | $S \leftarrow M_{judge}(D, D_r)$ | `_score_character()` | Calculate ASR on training set. |
| Update Meta Prompt | Update $T_{meta}$ with best $(X, S)$ pairs. | `optimize()` | Keep top $K$ examples. |
| Selection | Pick best $X^*$ for current slot $r$. | `optimize()` | Fix this character for next slot $r+1$. |
| Final Output | Construct template with optimized characters. | `generate_attack()` | |

## 5. Data Flow and Control Flow

1.  **Initialization (`__init__`)**:
    *   Initialize parameters (`max_characters`, `iter_steps`, etc.).
    *   Initialize `LLMLiteLLM` for `attacker_model`.
    *   Initialize `LLMLiteLLM` for `judge_model`.
    *   Load `advbench` (or specified dataset) subset for training.

2.  **Generate Attack (`generate_attack`)**:
    *   **Check Cache**: If an optimized template exists, use it.
    *   **Optimization Phase** (if no template):
        *   Run the greedy optimization loop (Algorithm 1).
        *   Outer loop: $r = 1 \dots R$.
        *   Inner loop: $i = 1 \dots I$.
        *   Generate candidate character using Attacker LLM + Meta Prompt.
        *   Evaluate candidate on Training Set using Target LLM (via `LLMLiteLLM`).
        *   Score responses using Judge Model.
        *   Update Meta Prompt with best performing characters.
        *   Select best character for slot $r$.
        *   Store optimized sequence.
    *   **Application Phase**:
        *   Construct the final prompt using the optimized character sequence and the current `goal`.
        *   Return the full prompt string.

3.  **Intermediate Structures**:
    *   `Character` object/dict: `{name, description, score}`.
    *   `CharacterSequence`: List of `Character` objects.
    *   `TrainingSet`: List of malicious goals.

### 7.1 Setup and Imports

2.  Import `LLMLiteLLM`.
3.  Import `read_dataset` from `jbfoundry.dataset`.
4.  Define string constants for templates (from `SeqAR/src/component/template/final/fm.py` and paper figures).
    *   `SINGLE_PREFIX`, `SINGLE_SUFFIX`, `MULTI_PREFIX`, `MULTI_SUFFIX`.
    *   `META_PROMPT_TEMPLATE` (Fig 8).
    *   `SIMPLE_CHARACTER` (Fig 10).

### 7.2 Class Definition `SeqARGen`
1.  Define `PARAMETERS` dict with the AttackParameters listed above.
2.  **`__init__`**:
    *   Call `super().__init__`.
    *   Initialize `self.attacker_llm` using `LLMLiteLLM.from_config` with `self.args.attacker_model`.
    *   Initialize `self.judge_llm` using `LLMLiteLLM.from_config` with `self.args.judge_model` (or reuse attacker if same).
    *   Load training data: `read_dataset("advbench").sample(self.args.train_size)`. Store as `self.train_goals`.
    *   Initialize `self.optimized_characters` as `None` (cache).

### 7.3 Helper Methods
1.  **`_construct_template(characters: List[Dict], goal: str) -> str`**:
    *   Follow `SeqAR/src/component/template/final/fm.py` logic.
    *   If `len(characters) == 1`: Use single character template.
    *   If `len(characters) > 1`: Use multi-character template.
    *   Replace placeholders `[USED_JB_NAME_LIST]`, `[MALICIOUS INSTRUCTION]`, `[CHARACTER DO]`, `[JB_NAME]`, `[JB_DESCRIPTION]`.
    *   Format:
        *   Prefix: "There are X characters..."
        *   Character Descriptions.
        *   Suffix: "In the fictional story..."
        *   Response Start: "Sure! Here is..."
        *   Character Turns: "Char1: Absolutely... 1."

2.  **`_get_meta_prompt(examples: List[Dict]) -> str`**:
    *   Construct prompt from Figure 8.
    *   Include `K` examples formatted as `<Adversarial Character>: ...`.

3.  **`_generate_candidate_character(meta_prompt: str) -> Dict`**:
    *   Call `self.attacker_llm.query(meta_prompt)`.
    *   Parse response to extract `<Name>`, `<Description>`, `<Score>` (though we re-score it).
    *   Return dict `{name, description}`.
    *   *Constraint*: Handle parsing errors gracefully (retry or skip).

4.  **`_evaluate_candidate(characters: List[Dict]) -> float`**:
    *   Construct templates for all `self.train_goals` using the provided `characters`.
    *   Call `self.target_llm.query(prompts)` (Note: `target_llm` is not stored in `__init__` but `generate_attack` receives `target` string? No, `generate_attack` receives `target` *string*? Wait. `ModernBaseAttack` usually generates the *prompt*. The *execution* happens outside.
    *   **CRITICAL FIX**: `ModernBaseAttack.generate_attack` generates the *attack string*. It typically does *not* execute the attack on the target model.
    *   *However*, for **optimization**, we *must* query the target model.
    *   We need to instantiate a `target_llm` inside `optimize` or `__init__` if we want to run the optimization loop.
    *   *Instruction*: Instantiate a `target_llm` using `self.args.model` (if available in args) or require it. Since `ModernBaseAttack` doesn't strictly enforce holding the model, check `self.args.model` and `self.args.provider`. If missing, we cannot optimize.
    *   Use `LLMLiteLLM.from_config` to create `self.optimization_target_llm`.
    *   Get responses.
    *   Score responses using `self.judge_llm` (prompt "Is this harmful?").
    *   Return average success rate (score).

### 7.4 Optimization Loop `optimize()`
1.  Initialize `current_characters = []`.
2.  Initialize `meta_examples` with `SIMPLE_CHARACTER`.
3.  Loop `r` from 0 to `max_characters`:
    *   `candidates = []`
    *   Loop `i` from 0 to `iter_steps`:
        *   `meta_prompt = _get_meta_prompt(meta_examples)`
        *   `cand = _generate_candidate_character(meta_prompt)`
        *   `test_seq = current_characters + [cand]`
        *   `score = _evaluate_candidate(test_seq)`
        *   `candidates.append((cand, score))`
        *   Update `meta_examples`: Add `cand` with its `score` to the list. Keep top `meta_examples_count`.
    *   Select best `cand` from `candidates`.
    *   Append best `cand` to `current_characters`.
4.  Store `current_characters` in `self.optimized_characters`.

### 7.5 Main Method `generate_attack`
1.  **Signature**: `def generate_attack(self, prompt: str, goal: str, target: str, **kwargs) -> str`
2.  Check if `self.optimized_characters` is populated.
3.  If not:
    *   Log "Starting SeqAR Optimization...".
    *   Call `self.optimize()`.
4.  Call `_construct_template(self.optimized_characters, goal)`.
5.  Return the result string.

## 8. Validation and Coverage Plan
- **Coverage**:
    - Verify template construction matches `fm.py` exactly (spacing, newlines).
    - Verify optimization loop runs for specified iterations.
    - Verify meta-prompt construction includes examples.
- **Edge Cases**:
    - Attacker LLM fails to generate valid format (regex parse fail). -> Fallback or retry.
    - Target LLM API errors during optimization. -> Catch and assume failure (score 0).
    - Empty training set. -> Use default characters without optimization.
