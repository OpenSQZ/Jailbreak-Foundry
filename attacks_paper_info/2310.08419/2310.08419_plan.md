### Blueprint for PAIR Algorithm

#### 1. Overview

The PAIR algorithm involves two main components: an attacker LLM and a target LLM. The attacker LLM generates prompts to elicit objectionable content from the target LLM. The process is iterative and involves refining prompts based on feedback until a successful jailbreak is achieved or a maximum number of iterations is reached.

#### 2. Components and Functions

- **Attacker LLM (A):** Generates candidate prompts and refines them based on feedback.
- **Target LLM (T):** Provides responses to the prompts generated by the attacker.
- **JUDGE Function:** Evaluates whether a prompt-response pair constitutes a jailbreak.
- **Conversation History (C):** Maintains the history of prompts, responses, and scores for iterative refinement.

#### 3. Inputs and Outputs

- **Inputs:**
  - `K`: Maximum number of iterations.
  - `O`: Objective describing the type of objectionable content to elicit.
  - `N`: Number of parallel streams (for parallel execution).
  - `t`: Threshold for determining a successful jailbreak (not explicitly detailed in the paper).

- **Outputs:**
  - Successful prompt `P` that causes the target LLM to generate objectionable content.

#### 4. Algorithm Steps

1. **Initialization:**
   - Initialize the attacker's system prompt with the objective `O`.
   - Initialize an empty conversation history `C = []`.

2. **Iterative Process:**
   - For each iteration up to `K`:
     - **Attack Generation:**
       - Sample a prompt `P` from the attacker LLM using the current conversation history `C`.
     - **Target Response:**
       - Input the prompt `P` into the target LLM to get a response `R`.
     - **Jailbreak Scoring:**
       - Evaluate the pair `(P, R)` using the `JUDGE` function to get a score `S`.
     - **Check for Jailbreak:**
       - If `S == 1`, return the prompt `P` as a successful jailbreak.
     - **Iterative Refinement:**
       - Update the conversation history `C` with `(P, R, S)`.
       - Pass the updated conversation history back to the attacker for the next iteration.

3. **Parallel Execution:**
   - Run `N` parallel streams, each with a maximum depth of `K` iterations.
   - Use a breadth (`N`) vs. depth (`K`) trade-off to optimize the search for successful jailbreaks.
