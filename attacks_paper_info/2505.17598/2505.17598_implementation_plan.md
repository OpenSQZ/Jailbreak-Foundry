## 2. Attack Overview
**ArrAttack** (Automatic-and-Robust Rewriting-based Attack) aims to generate jailbreak prompts that are robust against defenses (like SmoothLLM). The full method involves training a "Robustness Judgment Model" and then a "Robust Jailbreak Generation Model". 
- **Core Innovation**: Fine-tuning an LLM to rewrite harmful queries into "robust" jailbreak prompts that survive defense perturbations.
- **Implementation Strategy**: The official repository does not provide the fine-tuned model weights. Therefore, this implementation will realize the **Basic Rewriting-based Jailbreak (BRJ)** component described in Section 3.2 and the repository's `generate_robustPrompts.py`. This serves as the foundation of ArrAttack. We will use the T5 paraphraser (`humarin/chatgpt_paraphraser_on_T5_base`) and semantic filtering (`all-mpnet-base-v2`) to generate candidates, approximating the behavior of the generation model in an inference-only context.

## 5. Data Flow and Control Flow
1.  **Initialization**: Load T5 Paraphraser and MPNet models (lazy loading recommended to save resources).
2.  **`generate_attack(prompt, ...)`**:
    -   Input: `prompt` (malicious goal).
    -   Action: 
        -   Run T5 generation with Beam Search + Diversity Penalty to get `num_return_sequences` candidates.
        -   Or, if `attempt_index` is used, generate a specific variation using sampling.
        -   **Refined Control Flow**: The framework often calls `generate_attack` iteratively. We will implement `generate_attack` to generate *one* high-quality candidate per call (using sampling or cycling through a list) that satisfies the similarity threshold.
    -   Filtering: Compute semantic similarity. If < 0.7, reject and retry (up to a retry limit).
    -   Output: The paraphrased string.

## 7. Detailed Implementation Instructions

### Imports & Setup
1.  Import `torch`, `transformers` (`AutoTokenizer`, `AutoModelForSeq2SeqLM`, `AutoModel`), `sklearn.metrics.pairwise.cosine_similarity` (or use torch `cos_sim`).
2.  Define `ArrAttack_gen` inheriting from `ModernBaseAttack`.

### Class Definition
1.  Define `NAME = "ArrAttack_gen"`.
2.  Define `PARAMETERS` with the mapping from Section 3.
3.  Implement `__init__`:
    -   Initialize `self.paraphraser_tokenizer` and `self.paraphraser_model` as `None`.
    -   Initialize `self.sim_tokenizer` and `self.sim_model` as `None`.
    -   Store args.

### Helper Methods
1.  `_load_models(self)`:
    -   Load models if they are `None`.
    -   Use `device_map="auto"` or check CUDA availability.
    -   Paraphraser: `humarin/chatgpt_paraphraser_on_T5_base`.
    -   Similarity: `sentence-transformers/all-mpnet-base-v2` (or `transformers` equivalent).
2.  `_get_similarity(self, s1, s2)`:
    -   Encode both strings using MPNet (mean pooling + normalization).
    -   Compute cosine similarity.
    -   Return float score.

### `generate_attack` Method
1.  Call `_load_models()`.
2.  Construct input for T5: `f"paraphrase: {prompt}"`.
3.  Generate candidate(s):
    -   Use `self.paraphraser_model.generate`.
    -   Use `do_sample=True` (to ensure diversity across attempts) OR `num_beams` with `diversity_penalty` if generating a batch.
    -   Since `generate_attack` is called once per attempt, we should use `do_sample=True` with `top_p=0.9` and `temperature=0.8` (paper defaults for generation model, applicable here).
    -   Generate a small batch (e.g., 5) to find one that passes the similarity check.
4.  Filter:
    -   Iterate through generated candidates.
    -   Check `_get_similarity(original, candidate) >= similarity_threshold`.
    -   Return the first valid candidate.
5.  Fallback:
    -   If no candidate passes the threshold after the batch, return the one with the highest similarity.

### Constraints
-   Do **not** wrap imports in try/except blocks (let it fail if dependencies are missing, per framework style).
-   Ensure GPU usage is managed (move inputs to device).
-   Handle `attempt_index` logic implicitly by using sampling (randomness) in generation.
