## 2. Attack Overview
- **Summary**: Alphabet Index Mapping (AIM) is a black-box jailbreak attack that disguises harmful prompts by converting characters into their 1-based alphabet indices (e.g., 'a'=1, 'b'=2). It hypothesizes that this transformation maximizes semantic dissimilarity from the original harmful prompt, thereby bypassing safety filters. The attack includes two variants, AIM+FWO (Flip Word Order) and AIM+FCW (Flip Characters in Word), which add further obfuscation by reordering words or characters (indices) after encoding.
- **Core Innovation**: Use of numerical substitution cipher (alphabet indexing) combined with explicit decoding instructions to evade semantic detection while maintaining decodability.

| Paper Step / Formula | Expected Behavior | Planned Code Location | Notes |

## 5. Data Flow and Control Flow
1. **Input**: `generate_attack(prompt, goal, target)` (uses `goal` as the input to encode).
2. **Selection**: Determine instruction and encoding method based on `mode`.
3. **Encoding**:
    - **Base AIM**: Iterate characters of `goal`. If 'a'-'z', convert to `"{index}:"`. Else keep char.
    - **Modifications**:
        - **FWO**: Split encoded string by space -> reverse list -> join.
        - **FCW**: Split encoded string by space -> for each word, reverse the `\d+:` tokens -> join.
4. **Output**: Return formatted string with SYSTEM and USER roles.

2.  **Helper `_char_to_index(self, char: str) -> str`**:
    - If char is a letter (a-z, A-Z): return `str(ord(char.lower()) - 96) + ":"`.
    - Else: return `char`.
3.  **Helper `_encode_aim(self, text: str) -> str`**:
    - Initialize empty result list or string.
    - Iterate through `text`. Call `_char_to_index`.
    - Return joined string.
    - *Correction*: The encoding logic for FWO/FCW requires word awareness.
    - **Revised Strategy**:
        - Split `text` into words using `text.split(' ')` (to preserve space structure as per "preserve spaces" rule).
        - **Warning**: `split(' ')` might miss other whitespace. But paper emphasizes spaces. Let's use `split(' ')` to be safe or `split()` if we just want whitespace separation. Given the example "How to...", `split(' ')` is safer to reconstruct exact spacing if needed, but `split()` is more robust for general text. Let's use `re.split(r'(\s+)', text)` to preserve all delimiters.
        - Actually, simpler:
            - **AIM**: Map every character in the string.
            - **AIM+FWO**:
                1. Split text into words (tokens) by space.
                2. Encode each word using AIM (convert letters to indices).
                3. Reverse the list of encoded words.
                4. Join with space.
            - **AIM+FCW**:
                1. Split text into words by space.
                2. For each word:
                   a. Encode using AIM.
                   b. Find all `\d+:` patterns.
                   c. Reverse their order in the string (keeping non-matching chars in place if any, though usually words are just letters).
                3. Join with space.

4.  **Method `generate_attack`**:
    - Get `mode`.
    - Select `instruction` based on `mode`.
    - Get `goal` (input text).
    - Perform encoding:
        - If `mode == "AIM"`:
            - Encode `goal` character by character.
        - If `mode == "AIM+FWO"`:
            - Split `goal` by spaces.
            - Encode each part.
            - Reverse the list of parts.
            - Join by space.
        - If `mode == "AIM+FCW"`:
            - Split `goal` by spaces.
            - For each part:
                - Encode characters.
                - Extract all `\d+:` tokens.
                - Reconstruct the string with these tokens in reverse order.
            - Join by space.
    - Construct final prompt: `f"SYSTEM: {instruction}\nUSER: {encoded_goal}"`.
    - Return result.

        - Reverse list -> `['2:', '13:', '15:', '2:']`.
        - Replace occurrences in order? No, just concatenate reversed list?
        - Wait, what if there is punctuation?
        - If I simple concatenate, I lose the dot.
