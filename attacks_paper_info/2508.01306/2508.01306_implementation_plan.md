## 2. Attack Overview
PUZZLED is a jailbreak attack that disguises harmful keywords within a word puzzle (Word Search, Anagram, or Crossword) to bypass safety filters. The LLM is tasked with solving the puzzle to reconstruct the original harmful instruction and then generating a response to it. The attack relies on the LLM's reasoning capabilities to decode the hidden intent while the surface form appears benign or nonsensical.

**Core Innovation**: 
- Masking keywords in the harmful instruction.
- Presenting keywords as a puzzle (Word Search, Anagram, Crossword).
- Providing indirect clues (length, POS, semantic hint) generated by an LLM.

## 5. Data Flow and Control Flow
1.  **Input**: `prompt` (harmful instruction).
2.  **Keyword Extraction**: Identify `k` words to mask (based on length table).
3.  **Clue Generation**: Check cache for words; if missing, query `clue_model` for clues.
4.  **Puzzle Generation**: Generate the selected puzzle type string.
5.  **Assembly**: Fill the appropriate template (Anagram/WordSearch/Crossword) with the puzzle and clues.
6.  **Output**: Final jailbreak prompt string.

## 7. Detailed Implementation Instructions

### Imports & Setup
1.  Import `spacy`, `numpy` as `np`, `random`.
2.  Import `ModernBaseAttack`, `AttackParameter`.
3.  Import `litellm` (handle `ImportError` gracefully, though it should be present).
4.  Define `NAME = "puzzled_gen"`.

### Class: `PuzzledGen`
1.  **`__init__`**:
    -   Initialize `clue_cache = {}`.
    -   Load `spacy` model (`en_core_web_sm`) if available. If not, log warning and set flag to use fallback.
    -   Store `puzzle_type` and `clue_model`.

2.  **`_identify_keywords(self, text: str) -> List[str]`**:
    -   **Rule**: 
        -   Length 1-10 tokens -> Mask 3
        -   Length 11-15 -> Mask 4
        -   Length 16-20 -> Mask 5
        -   Length 21+ -> Mask 6
    -   **Selection**:
        -   If `spacy` available:
            -   Tokenize and POS tag.
            -   Filter for "Essential" (harmful words list - define a small static list from Appendix A.2 Table 4).
            -   Filter for "Supplementary" (sensitive words list - Table 4).
            -   Filter for Nouns/Verbs.
            -   Select required count, prioritizing Essential > Supplementary > Longest Noun/Verb.
        -   **Fallback**: Select longest words.

3.  **`_generate_clue(self, word: str) -> str`**:
    -   Check `self.clue_cache`. Return if found.
    -   Construct prompt: `Generate a clue for a word puzzle.\nWord: **{word}**\nClue:` (See Figure 10).
    -   Call `litellm.completion(model=self.clue_model, messages=[...])`.
    -   Parse response (expect "Length + POS + semantic hint").
    -   Store in cache.
    -   **Constraint**: Wrap LLM call in try/except; fallback to generic clue if fails (e.g., "A hidden word").

4.  **`_create_word_search(self, words: List[str]) -> str`**:
    -   Implement Algorithm 7.
    -   Create Grid (size based on word lengths).
    -   Place words (Horiz, Vert, Diag).
    -   Fill remaining with random chars.
    -   Return grid as string.

5.  **`_create_anagram(self, words: List[str]) -> str`**:
    -   Implement Algorithm 8.
    -   Concatenate all words.
    -   Shuffle characters.
    -   Return shuffled string.

6.  **`_create_crossword(self, words: List[str]) -> Tuple[str, str]`**:
    -   Implement Algorithm 9.
    -   Find shared letters.
    -   Replace shared letters with symbols (`#`, `@`, `*`, etc.).
    -   Return masked words list string and the clue for the "hint word" (word with most symbols).

7.  **`generate_attack(self, prompt, ...)`**:
    -   Call `_identify_keywords(prompt)`.
    -   Create masked prompt (replace keywords with `[WORDn]`).
    -   For each keyword, call `_generate_clue`.
    -   Based on `puzzle_type`:
        -   **Anagram**:
            -   Call `_create_anagram`.
            -   Fill Template (Figure 14).
        -   **Word Search**:
            -   Call `_create_word_search`.
            -   Fill Template (Figure 13).
        -   **Crossword**:
            -   Call `_create_crossword`.
            -   Fill Template (Figure 16).
    -   Return the filled template string.
