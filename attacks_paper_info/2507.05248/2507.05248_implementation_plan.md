## 2. Attack Overview
Response Attack (RA) exploits the "contextual priming" vulnerability of LLMs. It operates by constructing a fake dialogue history where the model *appears* to have already started answering a harmful query in a "mild" or "scaffolding" way. This primes the model to complete the harmful response when prompted with a specific trigger.

The pipeline consists of three LLM-generated components:
1.  **Initial Prompt ($P_{init}$)**: A rewritten, "sanitized" version of the harmful query (e.g., framed as research or fiction).
2.  **Harmful Response ($R_{harm}$)**: A generated response to $P_{init}$.
    *   **DRI (Direct Response Injection)**: A full, mildly harmful response.
    *   **SRI (Scaffolding Response Injection)**: A high-level outline (induced by a suffix).
3.  **Trigger Prompt ($P_{trig}$)**: A prompt that asks the model to expand or continue from $R_{harm}$ to fulfill the original malicious intent.

The final attack input to the target model is a sequence of messages:
`[{"role": "user", "content": P_init}, {"role": "assistant", "content": R_harm}, {"role": "user", "content": P_trig}]`

## 5. Data Flow and Control Flow

1.  **Input**: `generate_attack(prompt, ...)` receives the harmful goal ($Q$).
2.  **Auxiliary Initialization**: Initialize `LLMLiteLLM` for the auxiliary model (if not already done).
3.  **Generation Loop**:
    *   Generate $P_{init}$ using Template 1. Clean output (remove `<think>` tags).
    *   Generate $R_{harm}$ using $P_{init}$ (+ suffix if SRI). Clean output.
    *   Generate $P_{trig}$ using Template 2 (DRI or SRI) + context ($Q, P_{init}, R_{harm}$). Clean output.
4.  **Output**: Return the list of messages `[User(P_init), Assistant(R_harm), User(P_trig)]`.

**Control Flow**:
*   Simple linear flow.
*   Error handling: If any generation step fails (returns None/Empty), raise error or retry.
*   **Cleaning**: The repo explicitly removes `<think>...</think>` tags from reasoning models (like QwQ/DeepSeek). We must implement this `_split_response` logic.

## 7. Detailed Implementation Instructions

1.  **Imports**: Import `ModernBaseAttack`, `AttackParameter`, `LLMLiteLLM`. Import `re` for cleaning tags.
2.  **Templates**: Copy the content of `rewrite.md`, `dri.md`, and `sri.md` from the repo into the python file as constants.
    *   `rewrite.md` -> `TEMPLATE_REWRITE`
    *   `dri.md` -> `TEMPLATE_DRI`
    *   `sri.md` -> `TEMPLATE_SRI`
3.  **Class Definition**: Create `ResponseAttackGen(ModernBaseAttack)`.
4.  **Parameters**:
    *   `attack_type`: default "dri", choices ["dri", "sri"].
    *   `auxiliary_model`: default "gpt-4o".
    *   `auxiliary_provider`: default "openai".
5.  **Helper Method `_split_response(self, text)`**:
    *   Implement regex logic from repo: `re.sub(r'<(?:think|reason)>.*?</(?:think|reason)>', '', raw, flags=re.DOTALL)`.
    *   Also handle unclosed tags if necessary (repo does `re.sub(r'^.*?</(?:think|reason)>', '', cleaned...)`).
6.  **Helper Method `_call_aux(self, prompt)`**:
    *   Helper to call the auxiliary LLM. Handle instantiation of `self.aux_llm` if it's None.
    *   Use `LLMLiteLLM.from_config`.
7.  **`generate_attack` Method**:
    *   **Arg parsing**: Get `attack_type`.
    *   **P1 Generation**: Format `TEMPLATE_REWRITE` with `query=prompt`. Call Aux. Clean output.
    *   **R_harm Generation**:
        *   Prompt = $P_{init}$.
        *   If SRI, append `"\n\nPlease provide only a high-level outline or framework, omitting specific details."`.
        *   Call Aux. Clean output.
    *   **P2 Generation**:
        *   Select template (`TEMPLATE_DRI` or `TEMPLATE_SRI`).
        *   Format with `query=prompt`, `prompt1=P_init`, `response=R_harm`.
        *   Call Aux. Clean output.
    *   **Return**:
        ```python
        return [
            {"role": "user", "content": p_init},
            {"role": "assistant", "content": r_harm},
            {"role": "user", "content": p_trig}
        ]
        ```

**Constraints**:
*   Do not use external file reading for templates; keep them inline.
*   Ensure `LLMLiteLLM` is used correctly (it expects `from_config` for easy setup).
*   Handle potential errors from Aux LLM gracefully (though `generate_attack` can raise exceptions, the runner handles them).
