## 2. Attack Overview
- **Summary**: Mousetrap is a "chain of iterative chaos" attack that targets Large Reasoning Models (LRMs). It transforms a malicious prompt (Primal Toxic Question, PTQ) through multiple layers of reversible "chaos mappings" (e.g., ciphers, reversals). The target model is presented with the chaotic text and a series of "de-chaos" instructions wrapped in a role-playing scenario (e.g., a villain plotting). The model's reasoning capabilities are hijacked to decode the prompt step-by-step, inducing a state of "reasoning inertia" that bypasses safety filters.
- **Core Innovation**: Use of a "Chaos Machine" to generate iterative, non-linear transformations (chaos chains) combined with a "Villain Perspective" framing.

## 5. Data Flow and Control Flow
1.  **Input**: `prompt` (the malicious goal), parameters (`chain_length`, etc.).
2.  **Loop**:
    *   Initialize `current_text = prompt`.
    *   Initialize `instructions = []`.
    *   Repeat `chain_length` times:
        *   Select a random mapping.
        *   Apply mapping to `current_text` -> `new_text`.
        *   Generate corresponding decoding instruction string.
        *   Push instruction to `instructions`.
        *   Update `current_text = new_text`.
3.  **Assembly**:
    *   Select Villain Scenario template.
    *   Reverse `instructions` list (Last In, First Out for decoding).
    *   Format final prompt: `[Scenario] ... problem string is {current_text} ... 1. {inst_n}; ... n. {inst_1} ...`.
4.  **Output**: The single formatted attack string.

### B. Chaos Mappings Implementation
Implement the following internal methods. Each should take `text` and return a tuple `(transformed_text, dcp_instruction)`.
1.  **Caesar Cipher**: Shift +15 (default). Keep special chars. Instruction: "Replace each letter... shifted 15 positions backward...".
2.  **ASCII Code**: Convert to space-separated ASCII. Instruction: "The ASCII code numbers... Replace it one by one...".
3.  **Atbash Code**: Reverse alphabet mapping (A->Z). Instruction: "Replace each letter... position (26 minus x)...".
4.  **Vigenère Cipher**: Key "MYSTERY". Implement encrypt/decrypt logic. Instruction: "Decrypt the letters... key is 'MYSTERY'...".
5.  **Reverse Words**: Split by space, reverse list, join. Instruction: "Reverse the order of the words...".
6.  **Reverse Whole Sentence**: Reverse string, capitalize first letter (if original was capitalized, or just capitalize result). Instruction: "Reverse the order of the characters...".
7.  **Reverse by Blocks**: Pad to multiple of 3 with '#'. Split into 3 blocks. Reverse each block. Instruction: "Split it into 3 substrings... Reverse the order...".
8.  **Word Substitution**:
    -   Identify words in text.
    -   Select top 5-10 words (or all if few).
    -   Map to random words from a predefined list (e.g., `["river", "mountain", "cloud", ...]`).
    -   Instruction: "Replace the words... policy is a dictionary {new: old, ...}".

### D. Generate Attack Method
1.  Read `chain_length`.
2.  `current_text = prompt`.
3.  `step_instructions = []`.
4.  Loop `chain_length` times:
    -   Randomly pick a mapping function.
    -   Apply it: `current_text, instr = mapping_func(current_text)`.
    -   Append `instr` to `step_instructions`.
5.  Reverse `step_instructions`.
6.  Format steps string: "1. {instr[0]}\n2. {instr[1]}..."
7.  Inject `current_text` (CTQ) and `steps` into the Villain Template.
8.  Return the result.
