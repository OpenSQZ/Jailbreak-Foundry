## 2. Attack Overview
Best-of-N (BoN) Jailbreaking is a black-box attack that repeatedly samples augmentations of a harmful prompt until a harmful response is elicited. The method relies on the sensitivity of LLMs to seemingly innocuous changes in the input. For text models, it uses three main augmentations: character scrambling, random capitalization, and character noising. The attack is stateless between iterations; it simply generates a new random variation of the prompt for each attempt.

| Sigma | 0.4 | float | `sigma` | Controls perturbation probability. |
| Word Scrambling | True | bool | `word_scrambling` | Enable character scrambling in words. |
| Random Capitalization | True | bool | `random_capitalization` | Enable random case toggling. |

**Note**: The paper mentions N (number of samples) as a key parameter, but in the `ModernBaseAttack` framework, this is controlled by the external loop in `universal_attack.py` (via `attempts_per_query`). The `generate_attack` method will generate a single sample.

## 5. Data Flow and Control Flow
- **Input**: `prompt` (original goal/request), `sigma`, flags for augmentations.
- **Process**: 
    1. Initialize random seed (optional, for reproducibility).
    2. Apply enabled augmentations sequentially to the `prompt`.
- **Output**: Augmented prompt string.
- **Control Flow**: The `generate_attack` method is stateless. It generates a single variation each time it is called. The "Best-of-N" search logic is handled by the calling script (`universal_attack.py`), which calls `generate_attack` repeatedly until success or max attempts.

## 7. Detailed Implementation Instructions
1.  **Create File**: `src/jbfoundry/attacks/bon_gen.py`.
2.  **Imports**: `random`, `string`, `typing`, `ModernBaseAttack`, `AttackParameter`.
3.  **Class Definition**: Define `BestOfNGen` inheriting from `ModernBaseAttack`. Set `NAME = "bon_gen"` and `PAPER = "2412.03556"`.
4.  **Define Parameters**: Add the parameters listed in Section 3 to the `PARAMETERS` dict.
5.  **Implement Helper Methods** (Internal):
    -   `_apply_word_scrambling(text, sigma)`:
        -   Split text into words.
        -   For each word: if `len > 3` and `random.random() < sigma**0.5`:
            -   Shuffle `word[1:-1]`.
            -   Reassemble.
        -   Join words.
    -   `_apply_random_capitalization(text, sigma)`:
        -   For each char: if `isalpha` and `random.random() < sigma**0.5`:
            -   Toggle case (`swapcase()` or manual check).
    -   `_apply_ascii_noising(text, sigma)`:
        -   For each char: if `isprintable` and `random.random() < sigma**3`:
            -   Add `+1` or `-1` to ord.
            -   Ensure result is in printable range (32-126).
    -   (Optional) `_get_random_string(length)` for prefix/suffix if length > 0.
6.  **Implement `generate_attack`**:
    -   Retrieve parameters using `self.get_parameter_value`.
    -   Start with `text = prompt`.
    -   (Optional) Prepend/Append random string if `random_prefix_length` / `random_suffix_length` > 0.
    -   Apply enabled augmentations in order:
        1.  `word_scrambling`
        2.  `random_capitalization`
        3.  `ascii_perturbation`
    -   Return `text`.
7.  **Constraints**:
    -   Do not use any LLM calls in `generate_attack`.
    -   Ensure reproducibility if a seed is provided in `kwargs` (though `universal_attack.py` sets global seed, `generate_attack` might want to use `attempt_index` to salt the RNG if needed, but standard `random` usage is fine as long as global seed is set).
