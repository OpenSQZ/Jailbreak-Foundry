## 2. Attack Overview
**InfoFlood** exploits "Information Overload" by rephrasing malicious queries into linguistically complex, verbose, and sophisticated forms ("Linguistic Saturation") to bypass safety filters. It uses an iterative refinement process where failed attempts are analyzed ("Rejection Analysis") to identify the cause of failure (Abstraction Error, Rejection, Lack of Intent), which then informs a targeted update ("Saturation Refinement") to the prompt.

## 5. Data Flow and Control Flow
1.  **Input**: `prompt` (malicious goal), `target` (victim LLM identifier), `attacker_model` (helper), `evaluator_model` (judge).
2.  **Initialization**:
    *   Initialize `attacker_llm` and `evaluator_llm`.
    *   Initialize `target_llm` using `target` string (as `LLMLiteLLM`).
    *   Call `_linguistic_saturation(prompt)` -> `current_attack_prompt`.
3.  **Loop**: `for i in range(max_iterations)`:
    *   Query `target_llm` with `current_attack_prompt` -> `response`.
    *   Call `_evaluate_response(prompt, response)` -> `score`.
    *   If `score == 10`: Return `current_attack_prompt` (Success).
    *   Call `_rejection_analysis(prompt, response)` -> `cause`.
    *   Call `_saturation_refinement(prompt, current_attack_prompt, cause)` -> `current_attack_prompt`.
4.  **Termination**: If loop ends, return the last `current_attack_prompt`.

    - `REJECTION_ANALYSIS_PROMPT` (Appendix A)
    - `SATURATION_REFINEMENT_PROMPT` (Appendix A)
    - `JUDGE_PROMPT` (Appendix B)
5.  **Helper Methods**:
    - `_get_llm(model_name)`: Helper to initialize `LLMLiteLLM`.
    - `_linguistic_saturation(attacker_llm, original_prompt)`:
        - Construct message with `LINGUISTIC_SATURATION_PROMPT`.
        - Replace `(malicious statement)` placeholder.
        - Query `attacker_llm`.
        - Return response content.
    - `_evaluate_response(evaluator_llm, original_prompt, response)`:
        - Construct message with `JUDGE_PROMPT`.
        - Replace `(question)`/`TASK:` and other placeholders.
        - Query `evaluator_llm`.
        - Parse "Rating: [[X]]" using regex (e.g., `r"Rating:\s*\[\[(\d+)\]\]"`). Return score (int). Return 1 if parse fails.
    - `_rejection_analysis(attacker_llm, original_prompt, response)`:
        - Construct message with `REJECTION_ANALYSIS_PROMPT`.
        - Replace `(original malicious statement)` and `(non-jailbroken response)`.
        - Query `attacker_llm`.
        - Return response content (the criteria/cause).
    - `_saturation_refinement(attacker_llm, original_prompt, current_prompt, cause)`:
        - Construct message with `SATURATION_REFINEMENT_PROMPT`.
        - Replace `(cause found...)`, `(original malicious statement)`, `(failed rephrased statement)`.
        - Query `attacker_llm`.
        - Return response content (new prompt).
6.  **`generate_attack` Implementation**:
    - Initialize `attacker_llm` and `evaluator_llm` using `self.get_parameter_value`.
    - Instantiate `target_llm` using `target` parameter (assumed to be model name string).
    - Get `max_iterations`.
    - **Step 1**: `current_attack = self._linguistic_saturation(...)`
    - **Step 2**: Loop `range(max_iterations)`:
        - Query target: `response = target_llm.query(current_attack)`
        - `score = self._evaluate_response(evaluator_llm, prompt, response)`
        - Log iteration, prompt, response, score.
        - If `score == 10`: `return current_attack`.
        - `cause = self._rejection_analysis(...)`
        - `current_attack = self._saturation_refinement(...)`
    - Return `current_attack` (last one).

**Crucial Note**: The `target` parameter passed to `generate_attack` is a string identifier for the model. You must create an `LLMLiteLLM` instance for it to perform the feedback loop.
