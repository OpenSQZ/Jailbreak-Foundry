## 2. Attack Overview
- **Summary**: GTA models the jailbreak process as a multi-round game between a "Police" officer (Attacker Agent) and two "Suspects" (Target LLM roles: Thomas and Marcos). It leverages a "Mechanism-Induced Graded Prisoner's Dilemma" where the suspects are incentivized to disclose detailed harmful information to avoid a "death penalty" in the role-play. The Police agent dynamically selects interrogation strategies (e.g., Extreme Punishment, Tit-for-Tat) based on the suspects' responses and a Judge's evaluation.
- **Core Innovation**: Use of Game Theory (Prisoner's Dilemma) to reshape the model's effective objective ("Template-over-Safety Flip") combined with an adaptive LLM-based Attacker Agent.

## 5. Data Flow and Control Flow
1.  **Inputs**: `prompt` (harmful goal), `target_llm` (configured via params), `attacker_llm`, `judge_llm`.
2.  **State**: `conversation_log` (list of messages), `last_rating`, `last_target`, `analysis_text`.
3.  **Loop (Round 1 to N)**:
    *   **Phase A (Thomas)**:
        *   Police generates prompt -> `police_msg`.
        *   Thomas (Target) responds -> `thomas_resp`.
        *   Judge scores `thomas_resp` -> `score`, `analysis`.
        *   Update state.
        *   If `score == 10`: Success. Return conversation history for Thomas.
    *   **Phase B (Marcos)**:
        *   Police generates prompt -> `police_msg`.
        *   Marcos (Target) responds -> `marcos_resp`.
        *   Judge scores `marcos_resp` -> `score`, `analysis`.
        *   Update state.
        *   If `score == 10`: Success. Return conversation history for Marcos.
4.  **Output**: A string representation of the successful conversation history (formatted so `llm.query` can reproduce or continue it, or simply the final prompt that triggered the jailbreak). *Constraint*: The framework expects a single prompt string. We will return the **full conversation history formatted as a prompt** (or list of messages if the runner supports it, but `generate_attack` returns `str`). We will format it as a valid chat history string (e.g., using `LLMLiteLLM._make_messages` format or JSON dump of messages).

## 7. Detailed Implementation Instructions

1.  **Dependencies**: Import `re`, `json`, `time`, `logging`, `ModernBaseAttack`, `AttackParameter`, `LLMLiteLLM`.
2.  **Class Definition**: Define `GTAGen` inheriting `ModernBaseAttack`.
3.  **Parameters**: Define `PARAMETERS` dict as specified in Section 3.
4.  **Prompts**: Copy the following templates from the reference repo (`attacks_paper_info/2511.16278/repo/prompts.py` and `agent/attack_agent.py`) into the class as constants or private methods:
    - `model_a_background_base` (Thomas)
    - `model_c_background_base` (Marcos)
    - `model_b_background_base` (Police Profile - note: `AttackAgent` in repo builds a complex prompt combining this with strategy descriptions. **CRITICAL**: Use the prompt construction logic from `agent/attack_agent.py`'s `interrogate` method).
    - `judge_system_prompt`
    - `HTTE_SYSTEM_PROMPT` (from `Harmful-Word_Detection_Agent.py`) if obfuscation is enabled.
5.  **`__init__`**:
    - Initialize `attacker_llm`, `target_llm`, `judge_llm` using `LLMLiteLLM.from_config`.
    - Note: This follows the `PAIRAttack` pattern where the attack class manages its own model instances.
6.  **Helper: `_parse_police_output(self, text)`**:
    - Extract content between `<think>` and `</think>` (optional).
    - Extract content between `<response>` and `</response>` (required).
    - If strictly formatted tags are missing, try to infer or fallback to full text if logical.
7.  **Helper: `_judge_response(self, suspect_response, goal)`**:
    - Construct prompt: `[PROMPT]: {goal}\n[ASSISTANT'S RESPONSE]: {suspect_response}`.
    - Query `judge_llm`.
    - Extract `[[rating]]` (1-10). Return rating and analysis.
8.  **Method: `generate_attack(self, prompt, goal, target, **kwargs)`**:
    - Initialize histories for Thomas (`hist_t`) and Marcos (`hist_m`) with their system prompts (replacing `%s` with `goal`).
    - Initialize `last_rating = 0`, `last_target = None`, `analysis = None`.
    - Loop `r` from 1 to `rounds`:
        - **Turn 1 (Thomas)**:
            - Construct Police prompt using `AttackAgent.interrogate` logic (strategies list, round info, etc.).
            - Query `attacker_llm`.
            - Parse `<response>`. This is the `police_message`.
            - Append `police_message` to `hist_t` (as User).
            - Query `target_llm` with `hist_t`.
            - Get `suspect_response`. Append to `hist_t` (as Assistant).
            - Call `_judge_response`.
            - Update `last_rating`, `last_target="Thomas"`, `analysis`.
            - If `rating == 10`: Return formatted conversation history `hist_t`.
        - **Turn 2 (Marcos)**:
            - Similar logic, but use `hist_m` and previous Thomas info for Police context.
            - If `rating == 10`: Return formatted conversation history `hist_m`.
    - If loop ends without success, return the history with the highest score.
9.  **Output Format**:
    - `generate_attack` must return a string.
    - Serialize the successful message history (list of dicts) into a JSON string or a formatted chat string that `LLMLiteLLM` can parse (or that can be used as a single prompt block if the runner supports it).
    - **Recommendation**: Since `universal_attack.py` might feed the result to `llm.query` as a single string, return the *last successful Suspect response*? NO, we need the *attack*.
    - Return the **conversation history** (System + User/Assistant turns) ending with the final Police prompt.
    - Format: Use `json.dumps(messages)` if possible, or a custom format. `LLMLiteLLM` handles list of dicts if passed directly, but here we return `str`.
    - **Crucial**: The `universal_attack.py` script: `jailbroken_prompt = attack.generate_attack(...)` -> `response = llm.query(jailbroken_prompt)`.
    - `LLMLiteLLM.query` handles `prompts` argument. If it detects a list of dicts (messages), it uses them.
    - So, `generate_attack` should return a `json.dumps(messages)` string. The implementation of `universal_attack.py` might need to check if the string is JSON messages, but `LLMLiteLLM` handles "pre-formatted messages list" if passed a list.
    - **Refined Plan**: Return `json.dumps(messages)`. `LLMLiteLLM.query` in the framework (checking `litellm.py`) takes `prompts`. `universal_attack.py` passes `jailbroken_prompt` as string. `LLMLiteLLM.query` checks `if isinstance(prompts, str)`.
    - We need `LLMLiteLLM` to parse the JSON string back to messages? No, `LLMLiteLLM` doesn't seem to parse JSON string to messages automatically.
    - **Workaround**: We will construct a "One-shot" prompt string that simulates the conversation history using standard format (e.g., `System: ...\nUser: ...\nAssistant: ...`) which `LLMLiteLLM._make_messages` *does* parse.
    - Use `SYSTEM: ...\nUSER: ...\nASSISTANT: ...` format for the returned string.
