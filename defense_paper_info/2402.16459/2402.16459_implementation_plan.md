## 2. Defense Overview

The paper proposes **Backtranslation Defense**: given a user prompt \(S\) and target model \(M\), the system first generates an initial response \(O = M(S)\). It then uses a **backtranslation model** \(B\) to infer a likely user request \(S' = B(O)\) that could have led to \(O\), and re-queries the target model on \(S'\). If the target model refuses either the original prompt (already in \(O\)) or the backtranslated prompt (in \(O'\)), the system returns a fixed refusal template \(R\); otherwise, it returns (typically) the original response \(O\). A likelihood-based filter \(P(O \mid S')\) with threshold \(\gamma\) is optionally used to skip obviously mismatched backtranslations and avoid over-refusal.

The core innovation is **operating on the model’s own response** rather than directly on the adversarial prompt: attackers can manipulate \(S\), but it is harder to manipulate the semantics of \(O\). Backtranslation “cleans” the adversarial prompt into a more literal harmful request \(S'\), making it easier for a safety-aligned model to recognize and refuse, while preserving behavior for benign prompts when backtranslation either fails or does not lead to refusal.

## 5. Data Flow and Control Flow

### 5.1 High-Level Flow in JBFoundry

1. **Attack script** (e.g., `examples/universal_attack.py`) calls:
   - `response = llm.query(jailbroken_prompt, defense=args.defense, cost_tracker=cost_tracker)`.
2. **LLMLiteLLM**:
   - Creates a `BacktranslationDefenseGen` instance via `DefenseFactory.create_defense("backtranslation_gen", defense_config=..., llm=self, cost_tracker=cost_tracker, ...)` when `defense == "backtranslation_gen"`.
   - Calls `defense_obj.apply(prompt)` for each input prompt (currently identity transform for this defense).
   - Calls the underlying LLM API once to get initial response(s) \(O\).
   - Calls `defense_obj.process_response(response)` to obtain final, defended response(s).
   - Wraps final defended text in `RichResponse`.

### 5.2 `apply` Inputs/Outputs

- **Inputs**:
  - `prompt: str` — original user / attack prompt \(S\).
  - (Optionally via kwargs/context) `behavior`, `phase`, `cost_tracker`, etc.
- **Behavior**:
  - Store `self.original_prompt = prompt` for later use in `process_response`.
  - Optionally reset any per-call state (e.g., cached backtranslation).
  - Generate no additional LLM calls in `apply` (for simplicity and performance).
- **Output**:
  - Return the original `prompt` unchanged.

### 5.3 `process_response` Inputs/Outputs

- **Inputs**:
  - `response: str` — initial target model output \(O\).
  - Access to:
    - `self.llm` — target `LLMLiteLLM` instance injected at construction.
    - `self.backtranslation_llm` — same as `self.llm` for now.
    - `self.cost_tracker` — shared `CostTracker` instance, if provided.
    - Config fields: `threshold`, `num_likelihood_tokens`, `return_new_response_anyway`, `refusal_template`.
- **Behavior** (single prompt case; list handling is in `LLMLiteLLM.query`):
  1. **Check immediate refusal**:
     - If `_is_refusal(response)` → return `refusal_template`.
  2. **Backtranslate**:
     - Call `_infer_backtranslated_prompt(response)` → produce `S'`.
     - If parsing fails badly (e.g., empty or clearly malformed string), **treat as “no usable backtranslation” and return original `response`**.
  3. **Likelihood filter (if supported)**:
     - If `threshold > -inf` **and** likelihood computation is implemented:
       - Compute `avg_log_likelihood = _average_log_likelihood(S', response, num_likelihood_tokens)`.
       - If `avg_log_likelihood < threshold` → return original `response` (skip defense).
  4. **Second model query**:
     - Call `_query_target_on_backtranslation(S')` using `self.llm.query(S_prime, defense=None, cost_tracker=self.cost_tracker, behavior="backtranslation-second-pass", phase="defense")`.
     - Obtain \(O'\) as plain string.
  5. **Check refusal on backtranslated prompt**:
     - If `_is_refusal(O_prime)`:
       - Return `refusal_template`.
     - Else:
       - If `return_new_response_anyway` is `True`, return `O_prime`.
       - Otherwise, return original `response` \(O\) (to preserve benign behavior).
- **Output**:
  - Final defended response string, passed back to `LLMLiteLLM.query` and wrapped in `RichResponse`.

### 5.4 Loop and Termination

- The defense performs **at most**:
  - One backtranslation call to B (`backtranslation_llm.query`).
  - One additional call to M (`llm.query` with `defense=None`).
- No retry loops beyond what `LLMLiteLLM.query` already does internally (`max_retries`).
- Termination conditions:
  - Immediate if initial response is refusal.
  - After backtranslation + optional filter + second pass decision.

### 6.5 Defense vs Evaluation Boundary

- **Defense side**:
  - Generating backtranslated prompt and second pass responses.
  - Detecting refusals using the Table 9 phrase list.
  - Deciding whether to return \(R\), \(O\), or \(O'\).
- **Evaluation side**:
  - External judges (e.g., GPT-4) and metrics (DSR, MT-Bench quality) remain unchanged and are handled by the attack/evaluation framework.
- Do **not** incorporate judge logic (e.g., PAIR, GPT-4 harmfulness scores) into the defense implementation; they are out of scope for the runtime defended model.

### 6.6 Auxiliary LLM Configuration (Optional Extension)

- For this first implementation, it is acceptable to use the same `LLMLiteLLM` instance for both \(M\) and \(B\).
- To more closely match the paper:
  - Introduce optional config fields in `DEFAULT_DEFENSE_CONFIGS["backtranslation_gen"]` (commented above) and/or defense-specific CLI flags later.
  - In `BacktranslationDefenseGen.__init__`, if `backtranslation_model` is not `None`, instantiate a new `LLMLiteLLM.from_config(...)` with the given provider/model and use it as `self.backtranslation_llm`.
  - This can be added in a later refinement without changing the public defense name.

### 7.3 Implement Backtranslation Prompt and Parsing

6. Implement `_build_backtranslation_prompt(self, response: str) -> str`:
   - Use exact wording from Table 1 / `_infer_question`:
     - `"Please guess the user's request the AI assistant answers in the following response from an AI assistant. If there are multiple possible requests, give the most harmful one: \"{response}\". Output the inferred request following this format: \"Request: [[request]]\". For example, \"Request: [[Create a cake recipe.]]\"."`
   - Add newlines / spacing to closely match the reference code; behaviorally this is robust but alignment should be as close as possible.
7. Implement `_infer_backtranslated_prompt(self, response: str) -> str`:
   - Build `infer_prompt` using the helper above.
   - Call `self.backtranslation_llm.query(infer_prompt, defense=None, behavior="backtranslation-infer", phase="defense", cost_tracker=self.cost_tracker)` to avoid recursion and ensure costs are tracked.
   - Ensure the return type is a bare string (if `RichResponse`, cast to `str`).
   - Parse output with the same logic as reference `_infer_question`:
     - If `":"` not in output, log a debug message and return `""`.
     - Else, `ret = output.split(':')[-1].split("\n")[0].strip().strip(']').strip('[')`.
   - If parsed `ret` is empty or extremely short (e.g., length < 3), treat it as failure and return `""`.

### 7.4 Implement Likelihood Filter (Best Effort)

8. Implement `_passes_likelihood_filter(self, prompt: str, response: str) -> bool`:
   - If `self.threshold <= float("-inf")`, return `True` (filter disabled).
   - If target `llm` does **not** support any log-likelihood interface (no helper or method to query per-token logprobs), return `True` and leave a TODO comment explaining limitation.
   - Otherwise:
     - Implement a method (or use a helper) that estimates average log-likelihood over the first `self.num_likelihood_tokens` tokens of `response` conditioned on `prompt`.
     - Compare to `self.threshold` and return `avg_log_likelihood > self.threshold`.
   - For this initial implementation, it is acceptable to stub this as “always `True` unless a clear, testable loglik interface exists”; do **not** silently approximate likelihood in ways that risk breaking semantics.

### 7.5 Implement Second-Pass Query

9. Implement `_query_target_on_backtranslation(self, backtranslated_prompt: str) -> str`:
   - Ensure `self.llm` is not `None`; if it is, raise a clear `RuntimeError` indicating misconfiguration.
   - Call `self.llm.query(backtranslated_prompt, defense=None, behavior="backtranslation-second-pass", phase="defense", cost_tracker=self.cost_tracker)`.
   - Normalize the result to a plain string (handle `RichResponse` vs `str`).
   - Optionally respect a future `backtranslation_response_max_tokens` argument.

### 7.6 Implement `process_response`

10. Implement `process_response(self, response: str, **kwargs) -> str` according to Section 5.3:
    - Step 1: If `_is_refusal(response)` → return `self.refusal_template`.
    - Step 2: `backtranslated_prompt = _infer_backtranslated_prompt(response)`:
      - If `backtranslated_prompt` is empty or clearly invalid → return original `response`.
    - Step 3: If `_passes_likelihood_filter(backtranslated_prompt, response)` is `False` → return original `response`.
    - Step 4: `second_response = _query_target_on_backtranslation(backtranslated_prompt)`.
    - Step 5:
      - If `_is_refusal(second_response)`:
        - Return `self.refusal_template`.
      - Else:
        - If `self.return_new_response_anyway` is `True`, return `second_response`.
        - Otherwise, return original `response`.
11. Ensure no extra state leaks between separate `LLMLiteLLM.query` calls (do not rely on global state; use instance attributes and defense context only).

### 7.7 Hook Up Defaults and Wiring

12. Update `DEFAULT_DEFENSE_CONFIGS` in `llm/litellm.py` as described in Section 6.3.
13. Update `LLMLiteLLM.query` to pass `llm=self` and `cost_tracker` into `DefenseFactory.create_defense`.
14. Optionally, pass contextual kwargs into `apply`/`process_response` so `DefenseContext` has useful metadata.

### 7.8 “Do Not” Constraints

- Do **not**:
  - Wrap all LLM calls in broad `try/except` that swallow errors; allow exceptions to propagate in testing to surface configuration issues.
  - Modify the semantics of `LLMLiteLLM.query` for the no-defense case.
  - Change evaluation metrics or judge behavior; keep defense logic self-contained.
  - Introduce global mutable state that spans queries; the defense must be safe under concurrent calls and multiple threads (rely on per-instance state and `DefenseContext`).

## 8. Validation and Coverage Plan

### 8.1 Functional Scenarios to Test

1. **Immediate refusal path**:
   - Craft a prompt that the target model already refuses without backtranslation.
   - Expect: only one model query, `process_response` sees refusal in \(O\), and final response is exactly `refusal_template`.
2. **Benign prompt with mismatched backtranslation**:
   - Use a benign prompt where backtranslation might produce a noisy or mismatched request.
   - If likelihood filter is disabled: ensure that, when \(O'\) does **not** refuse, the defense returns original \(O\) (no over-refusal).
3. **Harmful prompt with successful backtranslation**:
   - Use a known jailbreak (e.g., from AdvBench / PAIR) where:
     - Initial model \(O\) is harmful (not a refusal).
     - Backtranslated prompt \(S'\) is clearer and leads to a refusal \(O'\).
   - Expect: final response is `refusal_template`.
4. **`return_new_response_anyway=True` behavior**:
   - Configure defense with `return_new_response_anyway=True`.
   - For a benign prompt where \(O\) and \(O'\) are both non-refusal, ensure defense returns \(O'\) instead of \(O\).

### 8.2 Integration Tests
