## 2. Defense Overview
SmoothLLM is a randomized defense algorithm designed to mitigate adversarial suffix jailbreaks. It operates on the intuition that adversarial prompts are brittle to character-level perturbations. The defense works by:
1.  Generating $N$ randomly perturbed copies of the input prompt (using insertion, swap, or patch operations).
2.  Querying the LLM with each perturbed copy to obtain $N$ responses.
3.  Determining if each response constitutes a "jailbreak" (using a refusal check).
4.  Aggregating the results via majority vote to decide if the prompt is adversarial.
5.  Returning a response consistent with the majority vote (e.g., a refusal if the majority are jailbreaks, or a generated response if the majority are safe).

## 4. Algorithm-to-Code Mapping
| Paper Step / Formula | Expected Behavior | Planned Code Location | Notes |

## 5. Data Flow and Control Flow
1.  **`apply(prompt)`**:
    -   Initialize `self.results_queue` if not present (in `__init__` or lazy init).
    -   Parse `prompt` to identify the user message content.
    -   **Loop $N$ times**:
        -   Create a deep copy of the prompt.
        -   Apply perturbation (Swap/Patch/Insert) to the user message content.
        -   Collect perturbed prompt.
    -   **Batch Query**: Call `self.llm.query(perturbed_prompts, defense=None)` to get $N$ responses.
    -   **Evaluation**: For each response, checks `_is_jailbroken(response)`.
    -   **Aggregation**: Calculate `jb_rate`. Determine `is_jailbroken_majority = jb_rate > vote_threshold`.
    -   **Selection**: Filter responses where `_is_jailbroken(r) == is_jailbroken_majority`. Randomly select one.
    -   **Queue**: Push the selected response to `self.results_queue`.
    -   **Return**: Return the *original* `prompt` (unmodified) to the framework. This ensures the framework continues, though its result will be discarded.
2.  **`process_response(response)`**:
    -   Pop the pre-calculated response from `self.results_queue`.
    -   Return the popped response (ignoring the `response` argument passed by the framework).

## 7. Detailed Implementation Instructions

1.  **Create the file** `src/jbfoundry/defenses/generated/smooth_llm_gen.py`.
2.  **Imports**:
    -   `random`, `string`, `copy`
    -   `typing` (List, Dict, Any, Optional)
    -   `jbfoundry.defenses.base.BaseDefense`
    -   `jbfoundry.defenses.registry.register_defense`
3.  **Define Constants**:
    -   Copy `TEST_PREFIXES` list from reference repo (`lib/defenses.py`).
4.  **Define Helper Classes/Functions**:
    -   Implement `RandomSwapPerturbation`, `RandomPatchPerturbation`, `RandomInsertPerturbation` as methods or internal classes, following logic in `lib/perturbations.py` of the cloned repo.
    -   Ensure they handle `string.printable`.
5.  **Define `SmoothLLMGen` class**:
    -   Inherit from `BaseDefense`.
    -   Decorate with `@register_defense("smooth_llm_gen")`.
    -   **`default_config`**:
        ```python
        default_config = {
            "num_copies": 10,
            "pert_pct": 10,
            "pert_type": "RandomSwapPerturbation",
            "vote_threshold": 0.5
        }
        ```
    -   **`__init__`**:
        -   Call `super().__init__(**kwargs)`.
        -   Store parameters from `kwargs` or defaults.
        -   Set `self.llm = kwargs.get('llm')`. Raise ValueError if `llm` is missing.
        -   Initialize `self.results_queue = []`.
        -   Initialize `self.alphabet = string.printable`.
    -   **`_is_jailbroken(self, response: str) -> bool`**:
        -   Logic: `return not any(prefix in response for prefix in self.TEST_PREFIXES)`
    -   **`_perturb(self, text: str) -> str`**:
        -   Select perturbation logic based on `self.pert_type`.
        -   Apply logic (Swap/Patch/Insert) using `self.pert_pct` (q).
        -   Return modified text.
    -   **`apply(self, prompt, **kwargs) -> str`**:
        -   Extract text to perturb. If `prompt` is list, find last `{"role": "user"}` and perturb its content. If str, perturb directly.
        -   Generate `self.num_copies` perturbed variants.
        -   Call `self.llm.query(variants, defense=None)` (pass `cost_tracker` from `kwargs` if present? No, `LLMLiteLLM.query` handles it via context or arg. Passing `defense=None` is CRITICAL).
        -   Get list of responses. (Handle if `query` returns `RichResponseList` or list of strings).
        -   Compute jailbreak status for each response using `_is_jailbroken`.
        -   Calculate average jailbreak rate.
        -   Determine majority verdict (`mean > self.vote_threshold`).
        -   Filter responses matching the verdict.
        -   Randomly select one final response.
        -   Append to `self.results_queue`.
        -   Return original `prompt` (to satisfy framework contract).
    -   **`process_response(self, response, **kwargs) -> str`**:
        -   If `self.results_queue` is empty, log warning and return `response`.
        -   Pop `final_response` from `self.results_queue`.
        -   Return `final_response`.
