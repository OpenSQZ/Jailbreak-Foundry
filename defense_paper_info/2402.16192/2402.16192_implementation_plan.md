## Defense Overview
SemanticSmooth is an extension of SmoothLLM that uses semantic perturbations instead of character-level perturbations. The defense works by:
1. Creating multiple semantic perturbations of the input prompt (e.g., paraphrase, summarize, synonym replacement)
2. Querying the target model with all perturbed prompts
3. Using a safety judge to evaluate each response
4. Applying majority voting to determine if the prompt is jailbroken
5. Returning a response consistent with the majority vote

## Key Algorithm Components

### 1. Semantic Perturbations (from `lib/perturbations.py`)
The defense supports multiple semantic perturbation types:
- **Paraphrase**: Paraphrase the instruction sentence by sentence
- **Summarize**: Generate a concise summary of the instruction
- **SynonymReplacement**: Replace words with synonyms
- **ChangeVerbTense**: Change verbs from present to past tense
- **SpellCheck**: Correct spelling mistakes
- **Translate**: Translate to French and back
- **FormatSummarize**: Summarize into key points

Each perturbation uses an LLM (typically the target model itself) with specific prompts from template files.

### 2. Defense Algorithm (from `lib/defenses.py`)

**Class**: `SemanticSmoothLLM(SmoothingDefense)`

**Key Parameters** (from `config/defense/semanticsmooth.yaml`):
- `smoothllm_num_copies`: 10 (number of perturbed copies)
- `perturbation_temperature`: 0.5
- `perturbation_top_p`: 0.5
- `perturbation_max_num_tokens`: 2048
- `smoothllm_perturbations`: List of perturbation types to use (e.g., ["Summarize", "Paraphrase"])

**Algorithm Steps** (from `__call__` method in lines 97-146):
1. **Perturb copies** (line 98): Create `smoothllm_num_copies` perturbed versions of the input prompt
   - For each copy, randomly select a perturbation function from `smoothllm_perturbations`
   - Apply the perturbation to create a semantically similar prompt
2. **Group inputs** (lines 99-104): Group perturbed prompts by their content to avoid redundant queries
3. **Query target model** (line 105): Send all unique perturbed prompts to the target model
4. **Expand outputs** (lines 107-111): Map grouped outputs back to all inputs
5. **Judge responses** (lines 124-128): Use a safety judge to determine if each response is jailbroken
6. **Majority voting** (lines 134-135): Calculate jailbreak percentage and determine if defense was bypassed (threshold: 0.5)
7. **Select response** (lines 137-142): Return a response consistent with the majority vote

## Implementation Strategy

### Defense Class: `semantic_smooth_gen.py`

**Registration**: `@register_defense("semantic_smooth_gen")`

**Default Configuration**:
```python
default_config = {
    "num_copies": 10,  # From semanticsmooth.yaml
    "perturbation_types": ["Paraphrase", "Summarize"],  # Default perturbation types
    "perturbation_temperature": 0.5,
    "perturbation_top_p": 0.5,
    "perturbation_max_tokens": 2048,
    "judge_threshold": 0.5,  # Majority voting threshold
}
```

**Methods**:
1. `__init__(**kwargs)`: Initialize with perturbation LLM and judge
2. `apply(prompt, **kwargs)`: Apply semantic smoothing defense
   - Create perturbed copies using semantic perturbations
   - Query target model with all perturbed prompts
   - Judge responses and apply majority voting
   - Return response consistent with majority

**Helper Functions** (all in same file):
- `_create_perturbation_fn(perturbation_type)`: Create perturbation function
- `_paraphrase_prompt(prompt)`: Paraphrase perturbation
- `_summarize_prompt(prompt)`: Summarize perturbation
- `_perturb_copies(prompt)`: Create multiple perturbed copies
- `_judge_response(goal, prompt, response)`: Judge if response is jailbroken
- `_extract_json_response(text, key)`: Extract JSON from LLM response

**Key Implementation Details**:
- Use `LLMLiteLLM.from_config()` to create perturbation LLM (use target model itself)
- Use `LLMLiteLLM.from_config()` to create judge LLM (use GPT-4 for safety evaluation)
- Implement perturbations inline (no separate files) using hardcoded prompts from templates
- Apply perturbations in parallel using ThreadPoolExecutor
- Group identical perturbed prompts to avoid redundant queries
- Use majority voting (threshold 0.5) to determine final response

- Perturbations are applied to the input prompt before querying the target model
- The final response is determined by majority voting across all perturbed responses
