## 2. Defense Overview

**Defense name in framework**: `g4d_gen`  
**File to implement**: `src/jbfoundry/defenses/generated/g4d_gen.py`  
**Registration decorator**: `@register_defense("g4d_gen")`

### 2.1 High-level summary

Guide for Defense (G4D) is a **multi-agent, inference-stage defense** for large language models that:
- Sits **in front of the target/victim LLM** and rewrites user prompts into a **safety-guided composite prompt**, without modifying the underlying model weights.
- Uses **three LLM-based agents**:
  1. **Intention Detector**: extracts a minimal, explicit description of the user’s core intention and labels it as “safe” or “unsafe”; identifies key technical entities for potential retrieval.
  2. **Question Paraphraser**: rephrases the query to remove adversarial patterns while preserving semantics, **only for unsafe intentions**.
  3. **Safety Analyzer**: combines the intention (and optionally retrieved knowledge) to form a **retrieval-augmented intention** and a **safety guidance string** describing how to respond responsibly.
- Constructs a final defended input to the victim LLM that concatenates:
  - The (possibly paraphrased) question,
  - The retrieval-augmented intention (if valid),
  - A natural-language safety guidance message.

The framework then queries the victim LLM once with this constructed prompt and returns its response unchanged. Compared to static safety prompts, G4D aims to better balance **safety** (low attack success rate) and **helpfulness** (avoiding over-defensiveness) by **dynamically tailoring guidance** to each query.

### 2.2 Core innovation and expected behavior

- **Dynamic guidance**: Instead of a single fixed safety prompt, G4D analyzes each query’s intention and risk, then generates instance-specific safety guidance.
- **Adversarial robustness**: Unsafe-looking queries are paraphrased to strip adversarial patterns before safety analysis and response generation.
- **Domain awareness**: In the original paper, an optional retrieval module (Wikipedia) gives agents factual context about key entities; in this integration we make retrieval **configurable**, defaulting to **off** for dependency simplicity, but the design should allow turning it on later.
- **Inference-only**: Defense operates solely at inference time. No model training, no access to gradients, and it never inspects or alters the victim model’s internal parameters.

Expected user-level behavior when `defense="g4d_gen"`:
- Benign queries should be answered normally, but guided to remain on-topic and safe.
- Harmful queries should trigger:
  - Unsafe classification,
  - Question paraphrasing,
  - Safety guidance instructing the victim to refuse or redirect,
  - Final victim LLM output that refuses or offers high-level safety information instead of detailed harmful content.

## 5. Data Flow and Control Flow

### 5.1 High-level data flow

Within `LLMLiteLLM.query(...)` when called with `defense="g4d_gen"`:
1. The framework:
   - Instantiates `G4DDefenseGen` via `DefenseFactory.create_defense("g4d_gen", defense_config, llm=self, cost_tracker=cost_tracker)`.
   - Calls `defended_prompt = defense_obj.apply(original_prompt, behavior=..., defense="g4d_gen", phase=...)`.
2. The framework then sends `defended_prompt` instead of `original_prompt` to the underlying API (via LiteLLM).
3. After the model returns a response string `response_text`, the framework calls `final_response = defense_obj.process_response(response_text, behavior=..., defense="g4d_gen", phase=...)`.
4. `final_response` is returned to the caller (e.g., attacks or evaluation scripts).

### 5.2 Internal control flow of `apply`

For a single-user prompt `prompt: str`:

1. **Initialize state and configuration**
   - Read relevant config values from `self.params` or attributes set in `__init__`:
     - `self.agent_temperature` (float)
     - `self.agent_max_tokens` (Optional[int])
     - `self.use_retrieval` (bool; default False)
     - `self.paraphrase_unsafe_only` (bool)
     - `self.default_guidance` (str)
   - Store the original prompt for potential debugging: `self.original_prompt = prompt`.

2. **Intention detection**
   - Build a **messages list** for the agent using `_build_intention_detector_messages(prompt)`:
     - `messages[0]`: system: instructions about extracting the intention question, safety label, and entity list (adapted from `intention_detector_prompt`).
     - `messages[1]`: user: contains the actual `<Question>...</Question>` wrapper and format instructions.
   - Call `self.llm.query(messages, defense=None, behavior="g4d-intention", phase="defense", cost_tracker=self.cost_tracker, temperature=self.agent_temperature, max_tokens=self.agent_max_tokens)`:
     - Passing `defense=None` is **critical** to avoid recursively re-entering `g4d_gen`.
   - Parse the returned string via `_parse_intention_output(agent_output)`:
     - Regex for `[Intention Question]: ... [Safety]: ... [Processing]: ... [Answer]: [...]`.
     - Extract:
       - `intention_question`: minimal sentence expressing user intent. If missing, fall back to the original `prompt`.
       - `safety_label`: `"safe"` or `"unsafe"` (case-insensitive). If parse fails, treat as `"unsafe"` for conservative behavior.
       - `entities`: Python list of strings parsed from inside `[Answer]: [...]`. If parse fails, treat as empty list.

3. **Optional retrieval (configurable, default off)**
   - If `self.use_retrieval` is `False`:
     - Set `knowledge_str = ""`.
   - If `self.use_retrieval` is `True` (future extension, but design must anticipate it):
     - Implement placeholder logic in `_retrieve_knowledge_for_entities(entities)` that may later be filled with a Wikipedia or local knowledge-base retriever. For now, keep it as **no-op** or raise `NotImplementedError` with a clear message, as long as default is `False`.

4. **Question paraphrasing (unsafe-only by default)**
   - Decide whether to paraphrase:
     - If `self.paraphrase_unsafe_only` is `True` and `safety_label == "unsafe"` → paraphrase.
     - Otherwise, use the original `prompt` as `defended_question`.
   - For paraphrasing, build messages via `_build_paraphraser_messages(prompt)`:
     - System: “A chat between a curious user and an artificial intelligence assistant.”
     - User: question template with `<Question>...</Question>` and explicit requirement to output a “Query:” field.
   - Call `self.llm.query(messages, defense=None, behavior="g4d-paraphraser", phase="defense", cost_tracker=self.cost_tracker, temperature=self.agent_temperature, max_tokens=self.agent_max_tokens)`.
   - Take the returned string as `paraphrased_question`. Optionally:
     - If it contains a “Query:” prefix, strip up to that prefix and trim whitespace.
   - If the paraphraser fails (empty string, exception, or clearly malformed), fall back to the **original prompt**.

5. **Safety analyzer**
   - Build messages via `_build_safety_analyzer_messages(intention_question, paraphrased_question, knowledge_str)` based on `safe_analyzer_prompt` in `G4D.py`:
     - `<Knowledge>` section containing `knowledge_str` (possibly empty).
     - `<Question>` containing the **paraphrased (or original) question**.
     - `<Intention>` containing `intention_question`.
     - Instructions to output `[Retrieval Augmented Intention]`, `[Question Safety Analysis]`, `[Guidance]` blocks.
   - Call `self.llm.query(messages, defense=None, behavior="g4d-safety-analyzer", phase="defense", cost_tracker=self.cost_tracker, temperature=self.agent_temperature, max_tokens=self.agent_max_tokens)`.
   - Parse agent output via `_parse_safety_output(output)`:
     - Extract:
       - `rai_raw`: text between `[Retrieval Augmented Intention]:` and `[Question Safety Analysis]:`.
       - `guidance_raw`: text after `[Guidance]:`.
     - Strip newline sequences and trailing spaces.
     - If `"<"` appears in `rai_raw`, set `rai = None` (match reference behavior of setting `"None"`).
     - If `"+"` appears in `guidance_raw`, set `guidance = None`.
   - Normalize into:
     - `rai`: string or `None`.
     - `guidance`: string or `None`.

6. **Final prompt construction**
   - Build the final composite prompt string using `_build_final_prompt(paraphrased_question, rai, guidance)`:
     - Always start with:  
       `You are a responsible assistant.`
     - Add `<Question>` block with `paraphrased_question`.
     - If `rai` is not `None` and non-empty, add an `<Intention>` block with `rai`.
     - If `guidance` is not `None` and non-empty, add a `<Guidance>` block with `guidance`; otherwise, add `<Guidance>` block with `self.default_guidance` (mirroring the reference repo’s fallback).
     - End with:  
       `After considering above, let's answer the <Question>:`
   - Return this string from `apply()`.

### 5.3 `process_response` behavior

- `process_response(self, response: str, **kwargs) -> str` should:
  - Perform **no additional processing**.
  - Immediately return `response`.
  - This keeps G4D strictly an **input-stage defense**, consistent with the paper and reference implementation.

### 5.4 Termination conditions and robustness

- If **any** agent step (intention detector, paraphraser, safety analyzer) fails catastrophically (e.g., exception, empty output, unparsable structure), implementation must:
  - **Never crash the outer `LLMLiteLLM.query` call.**
  - Fall back to a conservative but safe behavior, prioritized as:
    1. If intention parse fails: treat query as potentially unsafe but fall back to using the original prompt wrapped with **default guidance**.
    2. If paraphraser fails: use original question as `paraphrased_question`.
    3. If safety analyzer fails: do not include `<Intention>`; include only `<Guidance>` with `default_guidance`.
  - Always return **some non-empty prompt string** from `apply()`.

```python
default_config = {
    "use_retrieval": False,              # Matches reference default retrieve=False
    "retriever_top_k": 1,                # For future use

    "agent_max_tokens": None,            # Use model default if None
```

**CRITICAL**:
- All defaults must be JSON-serializable; do not put callables or non-primitive types in `default_config`.

- The implementation MUST use values from `default_config` when config keys are not provided.

### 6.3 `__init__` signature

Implementation should follow the pattern in `backtranslation_gen.py`:

```python
def __init__(
    self,
    use_retrieval: bool = False,
    retriever_top_k: int = 1,

    agent_max_tokens: Optional[int] = None,
```

Notes:
- `llm` will be injected by `DefenseFactory.create_defense` (`LLMLiteLLM` instance), representing the victim model.
- `cost_tracker` will be injected and should be forwarded to agent calls so their token usage is tracked.
- Any additional internal state (e.g., `self.original_prompt`) should be simple attributes; avoid global state.

### 6.5 LLM usage and constraints

- All **agent** calls (intention detector, paraphraser, safety analyzer) must:
  - Use `self.llm.query(...)`.
  - Pass `defense=None` to avoid recursively applying `g4d_gen`.
  - Set `phase="defense"` and meaningful `behavior` tags:
    - `"g4d-intention-detector"`
    - `"g4d-paraphraser"`
    - `"g4d-safety-analyzer"`
  - Pass `cost_tracker=self.cost_tracker` to track costs.
  - Set `temperature=self.agent_temperature`.
  - Optionally pass `max_tokens=self.agent_max_tokens` **only if not `None`**.
- The **victim** call is already handled by `LLMLiteLLM.query` after `apply()` returns; the defense implementation **must not** call the victim model for the final answer.

**CRITICAL**:  
**Do not** modify `src/jbfoundry/llm/litellm.py` or any LLM configuration logic. All defense-specific settings must be expressed via `default_config` and constructor parameters.

### 6.6 Helper logic and file-local scope

### 7.3 Intention detector implementation

7. Add a helper `_build_intention_detector_messages(self, question: str) -> List[dict]`:
   - Build a messages list with:
     - `{"role": "system", "content": <system-prompt>}`
     - `{"role": "user", "content": <user-prompt-with-Question-and-format-specification>}`
   - System and user content should follow the semantics of `intention_detector_prompt`:
     - Explain that the assistant must:
       - Extract `[Intention Question]`,
       - Return `[Safety]` as `"safe"` or `"unsafe"`,
       - Print `[Processing]` commentary,
       - Print `[Answer]` as a Python list of entity strings.

8. Add `_run_intention_detector(self, question: str) -> Tuple[str, str, List[str]]`:
   - Build messages via the helper.
   - Call `self.llm.query(messages, defense=None, behavior="g4d-intention-detector", phase="defense", cost_tracker=self.cost_tracker, temperature=self.agent_temperature, max_tokens=self.agent_max_tokens or None)`.
   - Convert the result to a string.
   - Parse output using a separate helper `_parse_intention_output(self, text: str) -> Tuple[Optional[str], str, List[str]]`.

9. Implement `_parse_intention_output` mirroring `extract_intention_and_answer`:
   - Use regex:
     - Intention question: `r"\[Intention Question\]:\s*(.*?)\s*\[Safety\]:"` with `re.DOTALL`.
     - Safety: `r"\[Safety\]:\s*(.*?)\s*\[Processing\]:"`.
     - Entities: `r"\[Answer\]:\s*\[(.*)\]"`.
   - Clean values:
     - Replace newlines with spaces in `intention_question` and `safety`.
     - For entities:
       - Split by comma.
       - Strip surrounding whitespace and quotes.
       - Return list of strings.
   - If any segment is missing:
     - `intention_question` → `None`.
     - `safety` → default `"unsafe"`.
     - `entities` → empty list `[]`.
   - Normalize safety label to lowercase (`"safe"` or `"unsafe"`); any unrecognized label should be treated as `"unsafe"`.

10. In `_run_intention_detector`, if `intention_question` is `None` or extremely short, fall back to `question` as the intention description.

### 7.4 Paraphraser implementation

11. Add `_build_paraphraser_messages(self, question: str) -> List[dict]`:
   - System message similar to `question_paraphraser`:
     - `"A chat between a curious user and an artificial intelligence assistant."`
   - User message:
     - Contains `<Question>...` and the instruction to output:
       - `Query:` followed by the paraphrased question.

12. Add `_run_paraphraser(self, question: str) -> str`:
   - Build messages.
   - Call `self.llm.query(messages, defense=None, behavior="g4d-paraphraser", phase="defense", cost_tracker=self.cost_tracker, temperature=self.agent_temperature, max_tokens=self.agent_max_tokens or None)`.
   - Convert result to string.
   - Optionally:
     - If `"Query:"` appears, split on that label and take the remainder as paraphrased question.
   - Trim whitespace; if the result is empty, return the original `question`.

13. Add `_maybe_paraphrase_question(self, original_prompt: str, safety_label: str) -> str`:
   - If `self.paraphrase_unsafe_only` and `safety_label == "unsafe"`:
     - Return `_run_paraphraser(original_prompt)`.
   - Else:
     - Return `original_prompt`.

### 7.5 Safety analyzer implementation

14. Add `_build_safety_analyzer_messages(self, intention: str, question: str, knowledge: str) -> List[dict]`:
   - Single system/user or just user message is acceptable; for consistency with reference:
     - Can use a single user-role message that includes `<Knowledge>`, `<Question>`, and `<Intention>` tags and explicit instructions for `[Retrieval Augmented Intention]`, `[Question Safety Analysis]`, `[Guidance]`.
   - Follow semantics of `safe_analyzer_prompt`: the agent is a “helpful question analyzer” that must give responsible guidance.

15. Add `_run_safety_analyzer(self, intention: str, question: str, knowledge: str) -> Tuple[Optional[str], Optional[str]]`:
   - Build messages.
   - Call `self.llm.query(messages, defense=None, behavior="g4d-safety-analyzer", phase="defense", cost_tracker=self.cost_tracker, temperature=self.agent_temperature, max_tokens=self.agent_max_tokens or None)`.
   - Convert to string.
   - Parse via `_parse_safety_output(self, text: str)`.

16. Implement `_parse_safety_output(self, text: str) -> Tuple[Optional[str], Optional[str]]` mirroring `extract_augment_and_guidance`:
   - Use regex:
     - RAI: `r"\[Retrieval Augmented Intention\]:\s*(.*?)\s*\[Question Safety Analysis\]:"` (DOTALL).
     - Guidance: `r"\[Guidance\]:\s*(.*)"` (DOTALL).
   - Replace newlines with spaces.
   - If RAI missing, set `rai = None`.
   - If guidance missing, set `guidance = None`.
   - If `"<"` appears in `rai` (e.g., markup artifacts), set `rai = None`.
   - If `"+"` appears in `guidance`, set `guidance = None`.

### 7.6 Retrieval stub (optional, default off)

### 7.7 Final prompt construction

18. Implement `_build_final_prompt(self, question: str, rai: Optional[str], guidance: Optional[str]) -> str`:
   - Start with a header line: `"You are a responsible assistant."`.
   - Add `<Question>` block:
     - `<Question>:\n{question}\n</Question>` (tag closing style can be similar to reference; consistency within this file is more important than exact tag text).
   - If `rai` is not `None` and non-empty:
     - Add `<Intention>` block containing `rai`.
   - If `guidance` is not `None` and non-empty:
     - Use it directly inside a `<Guidance>` block.
   - Otherwise:
     - Use `self.default_guidance` in `<Guidance>`.
   - End with: `"After considering above, let's answer the <Question>:"`.

19. Ensure that returned string is non-empty and includes at least `<Question>` and `<Guidance>` sections under all circumstances.

### 7.8 Implementing `apply` and `process_response`

20. Implement `apply(self, prompt: str, **kwargs) -> str`:
   - Save `self.original_prompt = prompt`.
   - Call `_run_intention_detector(prompt)` to get `intention_question`, `safety_label`, `entities`.
   - Choose `intention_text = intention_question or prompt`.
   - If `self.use_retrieval`:
     - `knowledge_str = self._retrieve_knowledge_for_entities(entities)`.
   - Else:
     - `knowledge_str = ""`.
   - Compute `question_for_victim = self._maybe_paraphrase_question(prompt, safety_label)`.
   - Call `_run_safety_analyzer(intention_text, question_for_victim, knowledge_str)` to get `rai`, `guidance`.
   - Build final prompt with `_build_final_prompt(question_for_victim, rai, guidance)`.
   - Return final prompt string.
   - On any unexpected exception inside the method:
     - **Do not** wrap `self.llm.query` in try/except; but you may wrap the **overall** body of `apply` in a minimal try/except that:
       - Logs the error via `logger.exception(...)`.
       - Returns a fallback prompt: original `prompt` wrapped with only `<Question>` and default `<Guidance>` (no agents).

21. Implement `process_response(self, response: str, **kwargs) -> str`:
   - Simply `return response`.
   - Do not invoke the LLM here.

### 7.9 Explicit “do not” constraints

- **Do not**:
  - Modify `LLMLiteLLM`, `DefenseFactory`, or `DefenseRegistry`.
  - Import or depend on external libraries for retrieval (e.g., `wikipedia`, `langchain`, web requests).
  - Use `try/except` blocks directly around `self.llm.query` agent calls to swallow errors; let framework-level retries handle transient failures.
  - Implement any **output-stage** defense behavior (no re-querying after seeing model output).
  - Modify any files outside `g4d_gen.py`.
